## ✅ 阶段 0 输出：能力映射 + 成功标准 + 交付清单（可写入简历的产出）

---

# 1. 能力清单（我们要证明的核心特质）

这些能力是你的“面试硬通货”，所有输出都要直接指向它们。

### A. **AI Native 思考**
- 能把复杂需求拆成“可执行的 AI 子问题”
- 能把“Prompt / Agent / 数据流”当作工具链设计系统
- 能通过 Prompt/Agent 迭代提升输出质量

### B. **第一性原理分析**
- 能把“行业/机会”拆成底层信号与变量（范式信号、风险假设）
- 能把不确定性变成“可验证假设 + 可量化指标”
- 能构建判断模型（范式转移判断 / 机会评估 / 风险评级）

### C. **系统化设计**
- 能把“情报 → 判断 → 决策 → 执行 → 复盘”做成可运行系统
- 能把成果抽象成“可复用流程 / 模板 / 工具”
- 能设计输入/输出规范 + 日志/复盘机制

### D. **协作能力（多角色/多 Agent）**
- 能设计“角色分工 + 协作链路 + 验收机制”
- 能让“多个数字员工”协同输出整体结果
- 能把协作结果可追踪、可评价、可复盘

### E. **可复用方法论**
- 能把流程打包成“可复制产品/模板”
- 能输出“下一次只换输入即可运行”的方法
- 能提供“案例+模板+执行指南”作为成果

---

# 2. 成功标准（怎么判断做对了）

### ✅ 成功标准 1：产出“面试/简历可写成果”
- 可以写进简历的成果（至少 3-5 条），形如：
  - “构建并实现了一个可运行的多 Agent + RAG 决策工作流，支持范式转移检测与机会评估。”
  - “搭建了一个可复用的 AI 分析体系，能够从信息流中自动生成可执行决策建议。”
- 这些成果要能回答：**我做了什么 → 价值是什么 → 证明了什么能力。**

### ✅ 成功标准 2：用 workflow 做真实评估（可复用验证）
- 你的 workflow 能接收一个真实输入（例如：一个游戏机会/趋势/团队），输出：
  - 评估结果（判定机会/风险/价值）
  - 决策建议（执行路径/优先级/资源配置）
- 这个输出可直接给面试官评估（“你能证明你会用 AI 做判断”）。

### ✅ 成功标准 3：方案可复用（可复制/可扩展）
- 你可以拿这套工作流去做第二个案例，只需替换输入即可；
- 你能给别人交付“说明 + 模板 + 角色配置”，别人可以复制执行。

---

# 3. 交付清单（我们要最终交出的成果包）

这些产出是你可写进简历、拿给面试官看的“案例交付包”。

### ✅ 交付 A：岗位能力映射报告（能力→成果对照）
- **形式**：结构化文档（1-2 页）
- **内容**：
  - 要证明的能力（上面能力清单）
  - 每个能力对应的产出（你要交付什么、怎么验证）
  - 简历写法模板（可直接复制）

---

### ✅ 交付 B：核心工作流结构图（AI 工作流系统设计）
- **形式**：架构图 + 说明（可用于 PPT/作品集）
- **内容**：
  - 多 Agent / 知识库 / RAG / 决策模块 / 日志模块
  - 输入/输出规范（数据来源、产物类型）
  - 关键路径（谁触发、谁计算、谁输出）

---

### ✅ 交付 C：真实评估案例输出（必须跑通一次）
- **形式**：一个“案例包”
- **内容**：
  - 输入（一个具体机会/项目/场景描述）
  - 输出（评估报告 + 决策建议 + 执行计划）
  - 评估模板（如何打分/验证输出）

---

### ✅ 交付 D：可复用 workflow 模板（工程化输出）
- **形式**：一套可复制的模板/Prompt/执行说明
- **内容**：
  - Agent 角色清单 + Prompt 模板
  - 输入/输出格式规范
  - 执行步骤 + 日志/复盘模板

---

### ✅ 交付 E：复盘报告 + 迭代计划
- **形式**：复盘文档（可写进作品集）
- **内容**：
  - 这套流程哪里有效、哪里需改进
  - 下次如何迭代（信号来源、评估逻辑、流程节点）
