## ✅ 阶段 0：核心计划（最核心的“能力清单 + 成功标准 + 交付清单”）

这是整个项目的“北极星”：  
**我们要证明的能力是什么？要交付什么成果？怎么判断是否达标？**

---

## 1️⃣ 能力清单（我们要显式证明这些能力）

### A. **AI Native 思考力**
- 能把复杂需求拆成“AI 能做的子问题”
- 能把“Prompt / Agent / 数据流”作为工具链构建系统
- 能通过 Prompt + Agent 协作把人类想法变成可执行流程

### B. **第一性原理分析能力**
- 能把“行业 / 项目 / 机会”拆成底层信号/变量/假设
- 能用“范式转移 + 机会判断”构建判断模型
- 能把不确定性拆解成可验证假设（并设计验证方式）

### C. **系统化设计能力**
- 能把“情报→判断→决策→执行→复盘”做成一个可运行系统
- 能构建“可复用的工作流模板 + 输入输出规范 + 日志/复盘机制”
- 能把项目输出转成可复用的流程/工具

### D. **协作能力（多角色多 Agent）**
- 能设计“角色分工+协作链路+验收机制”
- 能让多名数字员工（不同角色）协同完成一个整体结果
- 能让协作结果被清晰追踪、评价、复盘

### E. **可复用方法论（产品化思维）**
- 能把这套流程做成“可复制的产品/模板”
- 能输出“下一次只换输入即可复用”的方案
- 能用“案例+模板+执行指南”证明可复用性

---

## 2️⃣ 成功标准（怎么判断“做对了”）

### ✅ 成功标准 1：产出“面试/简历可写的成果包”
- 能写成 3-5 条简历成果（能力 + 产出 + 价值）
- 这些成果必须能直接回答 “你做了什么/怎么做的/结果是什么”

### ✅ 成功标准 2：能用 workflow 进行真实评估（可复用验证）
- 你搭建的 workflow 能接一个新输入（一个机会/项目/场景）  
- 并输出一套 “决策建议+评估结果+行动计划”
- 这个输出可直接让评审/面试官检验：你是否真的会用 AI 做决策

### ✅ 成功标准 3：方案可复用（可复制/可扩展）
- 你可以拿这套 workflow 去做第二个类似案例，只需替换输入即可
- 你能给别人交付 “操作说明 + 模板 + 角色配置”，别人能执行并产生类似结果

---

## 3️⃣ 交付清单（我们最终要输出的东西）

### 🎯 交付 A：岗位能力映射报告（核心“能力/成果”对照）
- 形式：结构化文档（1-2 页）
- 内容：
  - 能力清单 + 你如何用具体产出证明（举例）
  - 对应简历/面试亮点 phrasing（可直接复制写简历）

### 🎯 交付 B：核心工作流结构图（AI 工作流/系统设计）
- 形式：架构图 + 说明（可直接放进 PPT/作品集）
- 内容：
  - 多 Agent + 知识库 + RAG + 决策流 + 日志/复盘模块
  - 输入/输出是什么（谁给数据，谁产出结果）

### 🎯 交付 C：真实评估案例输出（必须可实际跑一次）
- 形式：一次“用 workflow 做评估”的完整产物包
- 内容：
  - 输入（一个具体机会/项目/场景描述）
  - 输出（评估报告 + 决策建议 + 执行计划 + 风险假设）
  - 评估结果如何打分（可复用评估模板）

### 🎯 交付 D：可复用 workflow 模板（工程化输出）
- 形式：一套可复制的模板/脚本/Prompt（像产品一样交付）
- 内容：
  - Agent 角色清单 + Prompt 模板
  - 输入/输出格式规范
  - 执行步骤/日志模板/复盘模板

### 🎯 交付 E：复盘与迭代计划
- 形式：复盘文档（这次流程问题、迭代建议、下一次怎么改）
- 内容：
  - 哪些环节有效？哪些需要改？
  - 下一阶段要重点优化什么？（例如信号来源、评估模型、协作方式）
