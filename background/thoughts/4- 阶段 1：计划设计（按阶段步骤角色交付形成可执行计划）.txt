## ✅ 阶段 1：计划设计（按阶段/步骤/角色/交付形成可执行计划）

这一步我们把阶段0的“能力/交付/成功标准”拆成**具体阶段 + 每阶段要做的事 + 由谁做 + 交付是什么 + 怎么验收**，并且给“招聘/生成团队”一个清晰的输入：  
“我们需要哪些专业的数字员工来完成每一步？”

---

## 1⃣ 阶段划分（五大阶段 + 核心目标）

| 阶段 | 目标 | 核心产出 |
|------|------|----------|
| 0. 需求澄清（已完成） | 明确能力清单 + 成功标准 + 交付清单 | 能力映射报告 + 交付清单 + 验收模板 |
| 1. 计划设计（本次输出） | 将交付拆成阶段步骤、定位所需能力与角色 | 阶段计划表 + 角色责任表 |
| 2. 团队构建（生成数字员工） | 根据计划创建专业数字员工（角色+Prompt+产出标准） | 角色画像 + Prompt 模板 + 创建流程 |
| 3. 执行交付（Workflow 运行） | 用数字员工执行计划并产出作品 | 评估案例输出 + 日志 + 复盘 |
| 4. 可复用包装（方法论产品化） | 把流程打包成可复制“产品/模板” | Workflow 模板 + 交付手册 |

## 2⃣ 阶段 1：计划设计输出（本次交付）

### 2.1 关键交付（本阶段必须产出）

#### 1) 阶段计划表（Stage Plan）
- 各阶段标题 + 目标
- 每阶段的“主要任务（Step）”
- 每个任务对应的“产出品（Deliverable）”
- 每个任务的“验收标准（How to know it works）”

#### 2) 角色责任表（Role Responsibility）
- 每个角色（数字员工）名称 + 核心职责
- 角色能力标签（例如：AI Prompt、第一性分析、系统设计、过程复盘）
- 该角色负责的任务/阶段
- 该角色交付的结果（格式/结构/模板）

#### 3) 计划对接“招聘团队”的输入（Recruitment Input）
- “这个任务需要什么能力” → “需要创建哪个角色”
- 每个角色的输出模板（prompt/产物格式/验收标准），可直接用 Skill 1/7/8 创建数字员工
- “角色协作关系图”（谁把结果交给谁、谁负责校验、谁负责复盘）

## 3⃣ 阶段 1 计划设计（详细内容）

### 阶段 1.1：能力转交付（拆解目标→产出→验收）

| 能力 | 产出形式 | 核心任务 | 验收标准 |
|------|----------|----------|----------|
| AI Native 思考 | Agent 流程图、Prompt 库 | 设计多 Agent 协作流程 | 能用同一输入跑出可评估结果 |
| 第一性原理分析 | 范式信号模型 + 评估矩阵 | 定义信号/指标 + 量化规则 | 输出可以打分的机会评估表 |
| 系统化设计 | Workflow 架构图 + 规范文档 | 设计输入/输出/日志/复盘机制 | 他人可按说明执行出结果 |
| 协作能力 | 角色分工表 + 协作流程图 | 定义角色/协作/验收 | 角色交付结果可追踪、可验收 |
| 可复用方法论 | 模板+手册 | 把流程做成可复制包 | 复制一次可运行，验证产出一致 |

### 阶段 1.2：阶段计划（按步骤展开执行路径）

#### ✅ 阶段 1：需求与目标确认（输出：能力映射报告）
- **任务 1.1：生成能力清单 & 成功标准**
- **任务 1.2：生成交付清单（可写简历的成果）**
- **验收**：这份文档能直接用来写简历成果，且能支撑“评估模板”输出。

#### ✅ 阶段 2：系统设计 & Workflow 规划（输出：结构图 + 规范）
- **任务 2.1：设计核心工作流（Agent / 知识库 / RAG / 决策）**
- **任务 2.2：定义每个模块输入/输出/责任**
- **验收**：任何人拿到 “系统架构图 + 输入说明”，即可复现一次评估输出。

#### ✅ 阶段 3：角色定义 & 团队构建（输出：角色画像 + Prompt 模板）
- **任务 3.1：定义需要的数字员工角色**
- **任务 3.2：为每个角色写 Prompt + 输出模板 + 验收标准**
- **验收**：这些角色可被 Skill 1/7/8 直接创建，并能“做出一个可检验产出”。

#### ✅ 阶段 4：执行 & 产出（输出：真实评估案例 + 日志）
- **任务 4.1：根据 workflow 运行一次“评估案例”**
- **任务 4.2：记录日志并完成复盘报告**
- **验收**：产出包含“评估结果 + 决策建议 + 行动计划”，供面试评估使用。

#### ✅ 阶段 5：产品化 / 可复用（输出：Workflow 模板 + 手册）
- **任务 5.1：将整个流程打包为“可复用模板”**
- **任务 5.2：生成“执行指南 + 角色指南 + 复盘指南”**
- **验收**：任何人可用模板重复一次评估，并输出完整结果。

## 4⃣ 阶段 1 - 角色责任（示例）

### 角色：需求澄清官（Capability Architect）
- **职责**：把岗位需求转成“能力清单 + 交付清单 + 成功标准”
- **输出**：
  - 能力映射文档
  - 交付清单（写简历用）
  - 验收模板（评估表）
- **交付形式**：结构化文档（兼可复制成简历成果）

### 角色：AI 工作流架构师（Workflow Architect）
- **职责**：设计多 Agent / 知识库 / 决策流程
- **输出**：
  - 系统架构图
  - Agent/模块分工表
  - 输入/输出规范
- **交付形式**：图+说明文档

### 角色：Prompt 设计师（Prompt Engineer）
- **职责**：把每个模块变成可执行的 Prompt / Agent
- **输出**：
  - Prompt 模板库
  - 跨模块数据格式
  - 结果校验模板
- **交付形式**：Prompt 文档 + 样例输出

### 角色：执行验证官（Execution Verifier）
- **职责**：运行一次 workflow、记录日志、输出评估成果
- **输出**：
  - 评估案例结果包
  - 执行日志（可写成作品）
  - 复盘报告（改进建议）
- **交付形式**：案例输出 + 复盘文档
