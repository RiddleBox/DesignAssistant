1.1 岗位能力清单（需要证明的核心特质）
AI Native 思考力：能把复杂问题拆解为可执行 Agent/Prompt/数据流
第一性原理分析：能从底层变量/信号抽象出“范式转移/机会”逻辑
系统化设计能力：把“情报→判断→决策→执行→复盘”做成一个可运行系统
协作能力（多角色协同）：能把任务拆分给“不同岗位/数字员工”并形成可观测结果
可复用方法论：你做的东西能够直接拿去复用、模板化、下一次快速启用
1.2 对应成功标准（怎么判断“做对了”）
产出可在简历/面试中直接呈现：你可以用“一个案例”清晰说明你做了什么、解决了什么、为什么是你做的。
能够用 workflow 做“真实评估”：你能通过这个 workflow 在一次模拟场景中输出一套决策/评估结果（而不是只有文档）。
你的 workflow 可复用：下一次换一个类似岗位/项目时，能复用这个结构、流程、prompt、模板。
✅ 交付清单（产出型输出，适合简历/面试展示）
下面这些产出，是我们要让“简历/作品集”里最核心的内容：

📌 交付品 A：岗位能力映射报告（Key-Ability Case）
内容：能力清单 + 你用什么产出/方法证明（每项能力都有对应的输出案例）
形式：1~2 页结构化文档（可作为“作品说明”）
📌 交付品 B：工作流结构图（AI 工作流 / 系统设计）
内容：多 Agent + 知识库 + RAG + 决策模块 + 日志/评价模块的整体架构图
形式：可直接用在简历/面试 PPT 的“架构图”
📌 交付品 C：实际案例（真实评估 / 验证流程演示）
内容：用 workflow 执行一次“真实评估”（比如：对一个游戏趋势/团队机会做一次完整评估）
形式：产出一个“评估输出包”（报告 + 评分表 + 关键决策建议）
📌 交付品 D：可复用workflow模板（工程化输出）
内容：包含 Agent 分工清单、数据输入/输出模板、Prompt模板、日志模板、验收模板
形式：一个“可直接复制的 workflow 交付包”（可作为 Github repo/项目结构）
📌 交付品 E：复盘报告 + 下一步迭代计划
内容：基于一次执行，写出“这套流程哪里有效、哪里需要改、下一步怎么迭代”
形式：用于展示“你有自驱复盘体系 + 持续改进能力”
