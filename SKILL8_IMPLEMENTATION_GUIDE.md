# Skill 8 实现指南

## 📌 重要说明

Skill 8是一个**AI驱动的对话式技能**，不需要编写传统的代码文件。所有功能都通过AI对话和系统工具调用来实现。

本文档提供了Skill 8的实现逻辑和使用方法。

---

## 🎯 核心实现逻辑

### 1. 项目需求分析功能

#### 1.1 模板库读取和解析

**实现方式**：使用 `read_file` 工具

**步骤**：
1. 读取模板索引：`read_file("data-layer/templates/index.json")`
2. 解析JSON数据，提取模板列表
3. 存储在对话上下文中供后续使用

**示例对话流程**：
```
AI：正在读取模板库...
[调用 read_file 工具]
AI：✓ 已加载3个系统内置模板和2个自定义模板
```

**错误处理**：
- 文件不存在 → 提示："模板库尚未初始化，请先使用Skill 7初始化模板系统"
- 文件格式错误 → 提示："模板库数据格式错误，请检查文件"

- 技术栈（关键词：前后端分离、全栈、微服务、云原生等）

检查以下必要信息是否完整：
- [ ] 项目类型
- [ ] 项目目标
- [ ] 项目规模
- [ ] 特殊要求

### 2. 匹配度计算和推荐策略选择

#### 2.1 模板匹配度计算算法

**实现方式**：AI分析 + 规则计算

**匹配度计算公式**：
```
匹配度 = 项目类型匹配(40%) + 描述相似度(30%) + 岗位覆盖度(20%) + 规模匹配(10%)
```

**详细计算逻辑**：

**1. 项目类型匹配（40分）**
```
规则：
- 完全匹配：40分
  例：用户需求"Web应用" vs 模板"软件开发团队"（适用于Web应用）
  
- 部分匹配：20分
  例：用户需求"游戏开发" vs 模板"软件开发团队"（有部分可复用岗位）
  
- 不匹配：0分
  例：用户需求"内容创作" vs 模板"软件开发团队"
```

**2. 描述相似度（30分）**
```
规则：基于关键词重叠度
- 提取用户需求关键词：[电商, 支付, 订单, Web, 前后端]
- 提取模板描述关键词：[Web, 移动应用, 前端, 后端, API]
- 计算重叠度：重叠关键词数 / 总关键词数
- 得分：重叠度 × 30

例：
用户关键词：[电商, 支付, 订单, Web, 前后端] (5个)
模板关键词：[Web, 移动应用, 前端, 后端, API] (5个)
重叠关键词：[Web, 前端, 后端] (3个)
重叠度：3 / 5 = 0.6
得分：0.6 × 30 = 18分
```

**3. 岗位覆盖度（20分）**
```
规则：模板岗位覆盖需求岗位的比例
- 分析用户需求，推断需要的岗位
- 检查模板是否包含这些岗位
- 计算覆盖率

例：
需求岗位：[产品经理, 前端, 后端, 测试, UI设计] (5个)
模板岗位：[产品经理, 前端, 后端, 测试, UI设计] (5个)
覆盖岗位：5个
覆盖率：5 / 5 = 1.0
得分：1.0 × 20 = 20分
```

**4. 规模匹配（10分）**
```
规则：团队规模与项目规模匹配度
- 小型项目（1-2月）：适合3-5人团队
- 中型项目（3-6月）：适合5-8人团队
- 大型项目（6月+）：适合8人以上团队

例：
用户项目：中型（3-6月）
模板团队：5人
匹配度：完全匹配
得分：10分
```

**匹配度计算示例**：
```
模板：软件开发团队
- 项目类型匹配：40分（完全匹配）
- 描述相似度：18分（60%重叠）
- 岗位覆盖度：20分（100%覆盖）
- 规模匹配：10分（完全匹配）
总分：88分（88%匹配度）
```

#### 2.2 推荐策略选择逻辑

**实现方式**：基于匹配度阈值的规则路由

**策略选择规则**：
```
IF 最高匹配度 >= 80%:
    策略 = "A：直接使用模板"
    执行 Step 3A
    
ELSE IF 最高匹配度 >= 50%:
    策略 = "B：组合多个模板"
    执行 Step 3B
    
ELSE:
    策略 = "C：AI自主分析"
    执行 Step 3C
```

**示例对话流程**：
```

- 软件开发团队：88%匹配 ✓

```

4. 优先级排序

- 保留：主策划（替代产品经理）
- 保留：系统策划、数值策划
- 移除：文案策划（职责合并到主策划）

- ⭐⭐：可选岗位，根据预算
- ⭐：辅助岗位，可外包或兼任

## 📝 总结

Skill 8是一个**纯AI对话驱动**的技能，不需要编写传统代码。所有功能都通过：

1. **AI对话分析**：理解用户需求，提取关键信息
2. **系统工具调用**：使用 `read_file`、`edit_file`、`replace_in_file` 等工具
3. **智能推荐生成**：基于模板库或AI分析生成团队配置
4. **多轮交互优化**：支持用户调整和反馈

**核心价值**：
- 降低团队搭建门槛
- 提供专业的团队配置建议
- 动态扩展知识库
- 与Skill 7无缝集成

**使用方式**：
用户直接与AI对话，描述项目需求，AI自动完成分析、推荐、保存、部署全流程。
