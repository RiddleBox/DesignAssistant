# Skill 7：快速团队模板系统

## 📋 技能概述

**技能名称**：快速团队模板系统  
**技能编号**：SKILL-007  
**版本**：v1.0.0  
**依赖**：Skill 1（数字员工招募与管理）、Skill 2（团队项目协作）  
**数据域**：`data-layer/templates/`（读写）

**核心功能**：
1. 提供预定义的团队模板（软件开发、内容创作、咨询等）
2. 支持用户自定义团队模板
3. 一键部署团队（批量创建员工 + 创建项目 + 分配成员）
4. 生成快速入门指南

**设计原则**：
- ✅ **领域无关**：提供通用模板，支持任意领域自定义
- ✅ **低耦合**：通过标准接口调用Skill 1和Skill 2
- ✅ **零依赖**：使用系统内置命令和AI工具
- ✅ **高效部署**：一键完成团队搭建，节省时间

---

## 🎯 使用场景

### 场景1：使用预定义模板快速搭建团队
```
用户："我想搭建一个软件开发团队"
→ 展示软件开发团队模板
→ 用户确认
→ 一键创建5名员工 + 1个项目 + 自动分配
→ 3分钟完成团队搭建
```

### 场景2：自定义团队模板
```
用户："创建一个游戏策划团队模板"
→ 引导用户定义岗位列表
→ 引导用户定义协作流程
→ 保存为自定义模板
→ 后续可重复使用
```

### 场景3：查看和管理模板
```
用户："查看所有可用的团队模板"
→ 展示预定义模板 + 自定义模板
→ 支持编辑、删除、导出模板
```

## 🔧 核心功能实现

### Step 0：环境检测与自动初始化

在开始任何操作前，必须先执行环境检测：

1. **检查模板数据层**：使用 `read_file` 检查 `data-layer/templates/index.json` 是否存在
2. **数据层不存在** → 执行零依赖自举初始化：
   - 告知用户："检测到模板数据层尚未初始化，正在自动创建..."
   - 使用 `terminal` 创建 `data-layer/templates/` 目录
   - 使用 `edit_file` 创建 `index.json` 文件
   - 使用 `edit_file` 创建3个预定义模板文件：
     - `tmpl_builtin_001.json`（软件开发团队）
     - `tmpl_builtin_002.json`（内容创作团队）
     - `tmpl_builtin_003.json`（咨询团队）

### Step 2C：使用模板部署团队

#### 阶段一：选择模板和收集信息

1. **检查用户是否指定了模板**
   - 已指定 → 继续
   - 未指定 → 展示模板列表，询问选择

2. **展示模板预览**：
   - 显示模板基本信息和成员配置
   - 询问："确认使用此模板部署团队吗？（是/否）"

3. **收集项目信息**：
   - 询问："请提供项目名称（默认：{模板名称}项目）"
   - 询问："请提供项目描述（可选）"

4. **询问是否自定义成员名称**：
   - "是否要自定义成员名称？（是/否，默认使用岗位名称）"
   - 如果是 → 逐个询问每个成员的名称

#### 阶段二：批量创建员工（调用Skill 1）

1. **告知用户开始部署**：
   ```
   🚀 开始部署团队...
   
   步骤1/3：创建团队成员（5人）
   ```

2. **遍历模板的成员配置**：
   - 对每个角色，调用Skill 1的"招聘员工"功能
   - 传递参数：岗位名称、职责列表、技能标签
   - Skill 1会自动创建员工工作空间和档案

3. **显示创建进度**：
   ```
   ✓ 创建产品经理（emp_20260212_001）
   ✓ 创建前端工程师（emp_20260212_002）
   ✓ 创建后端工程师（emp_20260212_003）
   ✓ 创建测试工程师（emp_20260212_004）
   ✓ 创建UI设计师（emp_20260212_005）
   
   步骤1完成：5名员工创建成功
   ```

#### 阶段三：创建项目（调用Skill 2）

1. **告知用户**：
   ```
   步骤2/3：创建项目工作空间
   ```

2. **调用Skill 2的"创建项目"功能**：
   - 传递参数：项目名称、项目描述、标签
   - Skill 2会创建项目工作空间和配置文件

3. **显示创建结果**：
   ```
   ✓ 创建项目：软件开发团队项目（proj_20260212_001）
   ✓ 创建项目工作空间
   
   步骤2完成：项目创建成功
   ```

#### 阶段四：分配成员到项目（调用Skill 2）

1. **告知用户**：
   ```
   步骤3/3：分配成员到项目
   ```

2. **遍历创建的员工列表**：
   - 对每个员工，调用Skill 2的"分配员工到项目"功能
   - 传递参数：员工ID、项目ID、权限级别（默认：读写）

3. **显示分配进度**：
   ```
   ✓ 分配产品经理到项目
   ```

### Step 2D：创建自定义模板

#### 阶段一：收集模板基本信息

1. **询问模板名称**：
   - "请提供模板名称（例如：游戏策划团队）"

2. **询问模板分类**：
   - "请提供模板分类（例如：游戏开发）"

3. **询问模板描述**：
   - "请简要描述模板用途（可选）"

#### 阶段二：定义团队成员

1. **询问成员数量**：
   - "请输入团队成员数量（1-10）"

2. **逐个定义成员**：
   - 对每个成员，询问：
     - 岗位名称
     - 职责列表（多条，用分号分隔）
     - 技能标签（多个，用逗号分隔）
     - 数量（默认1）

3. **示例交互**：
   ```
   定义第1个成员：
   - 岗位名称：主策划
   - 职责列表：游戏整体设计;玩法规划;团队协调
   - 技能标签：游戏设计,系统设计,团队管理
   - 数量：1
   
   定义第2个成员：
   - 岗位名称：系统策划
   - 职责列表：系统设计;数值平衡;功能规划
   - 技能标签：系统设计,逻辑思维,文档编写
   - 数量：1
   
   ...
   ```

#### 阶段三：定义协作流程（可选）

1. **询问是否定义协作流程**：
   - "是否要定义协作流程？（是/否，可后续编辑）"

2. **如果是，引导定义流程阶段**：
   - 询问阶段数量
   - 对每个阶段，询问：
     - 阶段名称
     - 参与角色
     - 输出物

#### 阶段四：保存模板

1. **生成模板ID**：
   - 格式：`tmpl_custom_NNN`
   - 示例：`tmpl_custom_001`

2. **创建模板配置文件**：
   - 使用 `edit_file` 创建 `data-layer/templates/模板ID.json`
   - 填充用户定义的数据

3. **更新模板索引**：
   - 使用 `replace_in_file` 在 `index.json` 中添加新模板记录
   - 更新统计数据：total +1, custom +1

#### 阶段五：反馈用户

```
✅ 自定义模板创建成功！

模板信息：
- 模板ID：tmpl_custom_001
- 模板名称：游戏策划团队
- 分类：游戏开发
- 成员数：4人
- 创建时间：2026-02-12 11:00:00

成员配置：
1. 主策划 × 1人
2. 系统策划 × 1人
3. 数值策划 × 1人
4. 文案策划 × 1人

下一步操作：
- 使用 "查看模板详情" 查看完整配置
- 使用 "使用模板部署团队" 快速搭建团队
- 使用 "编辑模板" 修改模板配置
```

### Step 2E：编辑模板

#### 阶段一：选择模板

1. **检查用户是否指定了模板**
   - 已指定 → 继续
   - 未指定 → 展示模板列表，询问选择

2. **检查模板类型**：
   - 系统内置模板 → 提示："系统内置模板不可编辑，可以基于此模板创建自定义模板"
   - 自定义模板 → 继续

#### 阶段二：选择编辑内容

1. **展示编辑选项**：
   ```
   请选择要编辑的内容：
   1. 基本信息（名称、分类、描述）
   2. 成员配置（添加/删除/修改成员）
   3. 协作流程
   4. 快速入门指南
   ```

2. **根据选择执行对应的编辑操作**

#### 阶段三：执行编辑

**编辑基本信息**：
- 询问要修改的字段
- 收集新值
- 使用 `replace_in_file` 更新模板配置文件

**编辑成员配置**：
- 展示当前成员列表
- 询问操作：添加/删除/修改
- 根据操作更新成员数组

**编辑协作流程**：
- 展示当前流程
- 询问操作：添加/删除/修改阶段
- 更新流程配置

#### 阶段四：保存并反馈

```
✅ 模板编辑成功！

更新内容：
- 修改了成员配置：添加了"关卡策划"角色
- 更新时间：2026-02-12 11:00:00

提示：
- 使用 "查看模板详情" 查看更新后的配置
- 后续使用此模板部署的团队将使用新配置
```

### Step 2F：删除模板

#### 阶段一：选择模板和确认

1. **检查用户是否指定了模板**
   - 已指定 → 继续
   - 未指定 → 展示模板列表，询问选择

2. **检查模板类型**：
   - 系统内置模板 → 提示："系统内置模板不可删除"
   - 自定义模板 → 继续

3. **安全确认**：
   - "确认要删除模板'{模板名称}'吗？此操作不可恢复。（输入模板名称以确认）"

#### 阶段二：执行删除

1. **删除模板配置文件**：
   - 使用 `terminal` 删除 `data-layer/templates/模板ID.json`

2. **更新模板索引**：
   - 使用 `replace_in_file` 从 `index.json` 中移除模板记录
   - 更新统计数据：total -1, custom -1

#### 阶段三：反馈用户

```
✅ 模板删除成功！

删除信息：
- 模板：游戏策划团队（tmpl_custom_001）
- 删除时间：2026-02-12 11:00:00

提示：
- 模板配置文件已删除
- 已使用此模板创建的团队不受影响
```

## 📋 工作准则

1. **数据访问规则**：
   - 只写 `data-layer/templates/` 目录
   - 只读其他数据层目录
   - 通过接口调用Skill 1和Skill 2

2. **模板设计原则**：
   - 预定义模板保持领域无关性
   - 支持用户自定义任意领域的模板
   - 模板配置清晰易懂

3. **部署流程规范**：
   - 先创建员工，再创建项目，最后分配成员
   - 每个步骤都要显示进度
   - 出错时要回滚已创建的资源

4. **错误处理**：
   - 模板不存在时，展示模板列表
   - 部署失败时，提示用户并回滚
   - 系统内置模板不可编辑和删除

5. **用户体验**：
   - 部署过程要显示进度条或步骤提示
   - 完成后生成快速入门指南
   - 提供清晰的下一步操作建议
