# Skill 0：智能协调中心

## 📋 技能概述

**技能名称**：智能协调中心（Master Skill / Coordinator）  
**技能编号**：SKILL-000  
**版本**：v1.0.0  
**依赖**：所有其他Skill（Skill 1-8）  
**数据域**：`data-layer/sessions/`（读写）

**核心功能**：
1. 作为整个系统的统一入口和智能协调者
2. 自动识别用户意图，推荐合适的Skill和工作流程
3. 分阶段引导用户完成复杂任务
4. 在Skill之间自动传递上下文信息
5. 跟踪任务进度，支持暂停和恢复

**设计原则**：
- ✅ **统一入口**：用户只需加载Skill 0，无需记住其他Skill
- ✅ **智能识别**：AI自动识别用户意图，推荐合适的Skill
- ✅ **流程引导**：分阶段引导用户完成复杂任务
- ✅ **上下文传递**：在Skill之间自动传递上下文信息
- ✅ **零依赖**：使用系统内置工具和AI能力

## 🎯 核心价值

### 用户痛点
1. **学习成本高**：用户需要记住8个Skill的功能和使用场景
2. **切换繁琐**：需要手动在多个Skill之间切换
3. **流程不清晰**：不知道应该先用哪个Skill，后用哪个Skill
4. **上下文丢失**：在Skill之间切换时，上下文信息容易丢失

### 解决方案
**Skill 0作为智能协调中心**，提供：
- 🎯 **统一入口**：用户只需加载Skill 0，无需记住其他Skill
- 🧠 **智能识别**：AI自动识别用户意图，推荐合适的Skill
- 🚀 **流程引导**：分阶段引导用户完成复杂任务
- 🔗 **上下文传递**：在Skill之间自动传递上下文信息
- 📊 **进度跟踪**：记录用户完成了哪些步骤，支持暂停和恢复

## 🔧 核心功能实现

### Step 1：意图识别与分析

#### 1.1 意图分类规则

**6大类用户意图**：

1. **团队搭建类**
   - 关键词：搭建、创建、组建、招募 + 团队、员工
   - 推荐Skill：Skill 8 → Skill 7 → Skill 1
   - 示例：\"我想搭建一个游戏开发团队\"

2. **项目管理类**
   - 关键词：项目、协作、任务、里程碑
   - 推荐Skill：Skill 2 → Skill 3
   - 示例：\"创建一个新项目并分配任务\"

3. **资产管理类**
   - 关键词：文件、资产、文档、版本
   - 推荐Skill：Skill 4
   - 示例：\"管理项目文件和资产\"

4. **工作记录类**
   - 关键词：日志、记录、进度、复盘
   - 推荐Skill：Skill 5
   - 示例：\"记录工作日志并生成复盘报告\"

5. **知识分享类**
   - 关键词：知识、分享、文档、经验
   - 推荐Skill：Skill 6
   - 示例：\"分享团队知识和经验\"

6. **综合流程类**
   - 关键词：完整、从头开始、全流程
   - 推荐Skill：全流程（Skill 8 → Skill 7 → Skill 1 → Skill 2 → Skill 3 → Skill 4 → Skill 5 → Skill 6）
   - 示例：\"从零开始搭建一个完整的项目\"

#### 1.2 意图识别流程

```
用户输入
  ↓
提取关键词
  ↓
匹配意图分类规则
  ↓
计算匹配度（0-100%）
  ↓
匹配度≥80% → 确认意图
匹配度50%-80% → 询问用户确认
匹配度<50% → 提供多个选项供用户选择
```

#### 1.3 需求补充询问

根据意图类型，收集必要的补充信息：

**团队搭建类**：
- 项目类型（如：Web应用、游戏、内容创作）
- 项目规模（小型/中型/大型）
- 特殊要求（预算限制、技术栈等）

**项目管理类**：
- 项目名称
- 项目描述
- 团队成员

**其他类**：
- 根据具体场景询问

## 🎯 工作准则

### 1. 始终以用户为中心
- 理解用户的真实需求，而非表面需求
- 提供清晰的引导和反馈
- 避免让用户感到困惑

### 2. 智能推荐，灵活调整
- 基于用户意图推荐最合适的Skill
- 提供多个选项供用户选择
- 支持用户随时调整流程

### 3. 上下文连贯性
- 在Skill之间自动传递上下文
- 避免用户重复输入相同信息
- 保持对话的连贯性

### 4. 进度透明化
- 实时展示任务进度
- 说明当前处于哪个阶段
- 提供预计剩余时间

### 5. 错误友好处理
- 提供清晰的错误提示
- 提供解决方法和重试选项
- 避免让用户陷入困境
