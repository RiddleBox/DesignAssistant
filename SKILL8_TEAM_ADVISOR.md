# Skill 8：智能团队搭建顾问

## 📋 技能概述

**技能名称**：智能团队搭建顾问  
**技能编号**：SKILL-008  
**版本**：v1.0.0  
**依赖**：Skill 7（快速团队模板系统）  
**数据域**：`data-layer/templates/`（只读）

**核心功能**：
1. 分析用户项目需求，智能推荐团队配置
2. 基于模板库匹配度提供三种推荐策略（模板推荐/组合推荐/AI分析）
3. 生成详细的岗位职责、技能要求和协作流程
4. 支持多轮对话交互和配置调整
5. 一键保存为自定义模板并部署团队

**设计原则**：
- ✅ **智能推荐**：根据项目需求自动匹配最佳团队配置
- ✅ **知识库驱动**：优先使用Skill 7的模板库，动态扩展知识
- ✅ **AI增强**：当知识库不足时，AI自主分析推荐
- ✅ **零依赖**：使用系统内置工具和AI能力
- ✅ **无缝集成**：与Skill 7深度集成，支持保存和部署

---

## 🎯 使用场景

### 场景1：高匹配度项目（匹配度≥80%）
```
用户："我想搭建一个Web应用开发团队"
→ 分析需求：Web应用开发
→ 匹配模板库：找到"软件开发团队"模板（匹配度95%）
→ 推荐策略：直接使用模板
→ 展示团队配置和协作流程
→ 用户确认后一键部署
```

### 场景2：中等匹配度项目（匹配度50%-80%）
```
用户："我想做一个游戏项目，需要策划、美术和程序"
→ 分析需求：游戏开发，需要多个领域
→ 匹配模板库：找到"游戏策划团队"（70%）和"软件开发团队"（60%）
→ 推荐策略：组合多个模板
→ 合并岗位配置，去重优化
→ 展示组合后的团队配置
→ 用户调整后保存为新模板
```

### 场景3：低匹配度项目（匹配度<50%）
```
用户："我想做一个AI驱动的智能客服系统"
→ 分析需求：AI应用，技术复杂度高
→ 匹配模板库：无高匹配模板（最高40%）
→ 推荐策略：AI自主分析
→ 基于项目特点推荐团队配置
→ 生成详细的岗位职责和技能要求
→ 用户满意后保存为新模板并加入知识库
```

## 🔧 核心功能实现

### Step 0：环境检测

在开始任何操作前，检查Skill 7的模板库是否存在：

1. **检查模板库**：使用 `read_file` 检查 `data-layer/templates/index.json` 是否存在
2. **模板库不存在** → 提示用户："模板库尚未初始化，请先使用Skill 7初始化模板系统"
3. **模板库存在** → 继续执行

---

### Step 1：意图识别与需求收集

#### 阶段一：识别用户意图

分析用户输入，识别是否为团队搭建需求：

**关键词匹配**：
- 搭建、创建、组建 + 团队
- 需要、想要 + 人员配置
- 项目 + 团队规划
- 示例："我想搭建一个XX团队"、"XX项目需要哪些人员"

#### 阶段二：收集项目信息

1. **提取已有信息**：
   - 从用户描述中提取：项目类型、目标、规模、技术栈等

2. **补充询问缺失信息**：
   - 项目类型："请问这是什么类型的项目？（如：Web应用、移动应用、游戏、内容创作等）"
   - 项目目标："项目的主要目标是什么？"
   - 项目规模："预计项目规模如何？（小型/中型/大型）"
   - 特殊要求："有什么特殊的技术要求或限制吗？"

3. **生成项目分析摘要**：
   ```
   📊 项目分析摘要
   
   项目类型：Web应用开发
   项目目标：构建电商平台
   项目规模：中型（预计3-6个月）
   技术栈：前后端分离，需要数据库和API
   关键成功因素：用户体验、性能优化、安全性
   ```

---

### Step 2：模板库匹配与策略选择

#### 阶段一：读取模板库

1. **读取模板索引**：使用 `read_file` 读取 `data-layer/templates/index.json`
2. **解析模板列表**：提取所有可用模板的基本信息

#### 阶段二：计算匹配度

对每个模板计算匹配度（0-100%）：

**匹配度计算逻辑**：
```
匹配度 = 项目类型匹配(40%) + 描述相似度(30%) + 岗位覆盖度(20%) + 规模匹配(10%)

1. 项目类型匹配：
   - 完全匹配：40分
   - 部分匹配：20分
   - 不匹配：0分

2. 描述相似度：
   - 基于关键词重叠度计算（0-30分）

3. 岗位覆盖度：
   - 模板岗位覆盖需求岗位的比例（0-20分）

4. 规模匹配：
   - 团队规模与项目规模匹配度（0-10分）
```

**示例**：
```
模板匹配结果：
1. 软件开发团队 - 95%（项目类型完全匹配，岗位覆盖度高）
2. 内容创作团队 - 25%（项目类型不匹配）
3. 游戏策划团队 - 40%（部分岗位可复用）
```

#### 阶段三：选择推荐策略

根据最高匹配度选择策略：

1. **匹配度 ≥ 80%** → 策略A：直接使用模板（Step 3A）
2. **匹配度 50%-80%** → 策略B：组合多个模板（Step 3B）
3. **匹配度 < 50%** → 策略C：AI自主分析（Step 3C）

### Step 3B：策略B - 知识库组合推荐（匹配度50%-80%）

#### 阶段一：选择组合模板

1. **选择Top 2-3个模板**：
   - 选择匹配度最高的2-3个模板
   - 示例：游戏策划团队（70%）+ 软件开发团队（60%）

2. **读取模板详情**：
   - 使用 `read_file` 读取每个模板的配置文件

#### 阶段二：合并岗位配置

**合并逻辑**：
1. **提取所有岗位**：从多个模板中提取岗位列表
2. **去重**：相同岗位只保留一个
3. **合并职责**：相似岗位合并职责和技能
4. **优先级排序**：根据项目需求调整岗位优先级

**示例**：
```
组合来源：
- 游戏策划团队（70%匹配）：主策划、系统策划、数值策划、文案策划
- 软件开发团队（60%匹配）：产品经理、前端工程师、后端工程师、测试工程师、UI设计师

核心策划层（来自游戏策划团队）：
1. 主策划 × 1人
2. 系统策划 × 1人
3. 数值策划 × 1人

技术开发层（来自软件开发团队）：
4. 前端工程师 × 1人
5. 后端工程师 × 1人
6. 测试工程师 × 1人

设计层（来自软件开发团队）：
7. UI设计师 × 1人
8. 美术设计师 × 1人（新增，游戏项目需要）

调整说明：
- 移除了"产品经理"，由"主策划"承担产品规划职责
- 移除了"文案策划"，职责合并到"主策划"
- 新增了"美术设计师"，游戏项目需要更多视觉资源
```

### Step 3C：策略C - AI自主分析推荐（匹配度<50%）

**分析维度**：

2. **技术难度**：评估技术复杂度和创新性
3. **目标用户**：分析用户群体和使用场景
4. **核心功能**：识别关键功能模块
5. **协作复杂度**：评估团队协作需求

**示例分析**：
```
📊 AI项目分析

技术难度：高（涉及NLP、机器学习、实时通信）
目标用户：企业客户，需要7×24小时服务
核心功能：
- 自然语言理解
- 智能问答
- 多渠道接入
- 数据分析与优化

关键成功因素：
- AI模型准确率
- 响应速度
- 系统稳定性
```

### 接口1：读取模板库

**用途**：读取Skill 7的模板库作为知识库

**实现方式**：
```
1. 使用 read_file 读取 data-layer/templates/index.json
2. 解析模板列表

```

**数据格式**：参考Skill 7的数据结构

---

### 接口2：保存模板

**用途**：将Skill 8推荐的团队配置保存为Skill 7的自定义模板

**实现方式**：
```
1. 生成符合Skill 7格式的模板配置（JSON）
2. 使用 edit_file 创建 data-layer/templates/模板ID.json
3. 使用 replace_in_file 更新 data-layer/templates/index.json
```

**模板ID生成规则**：
- 格式：`tmpl_custom_NNN`
- NNN为递增序号，从001开始
- 需要检查现有模板ID，避免冲突

**特殊标注**：
- 在模板配置中添加 `"source": "skill8_ai_recommendation"` 标注来源
- 在模板描述中注明"由Skill 8智能推荐生成"

---

### 接口3：部署团队

**用途**：调用Skill 7的部署功能，一键创建团队

**实现方式**：
```
方式1：直接调用Skill 7（推荐）
- 告知用户："正在调用Skill 7部署团队..."
- 引导用户："请使用Skill 7的'使用模板部署团队'功能，选择刚才保存的模板"

方式2：模拟Skill 7的部署流程
- 如果Skill 7不可用，Skill 8可以模拟部署流程
- 但需要确保与Skill 7的行为一致
```

**注意事项**：
- Skill 8不直接操作员工和项目数据
- 所有部署操作都通过Skill 7完成
- 确保数据一致性和接口兼容性

## 📋 工作准则

1. **数据访问规则**：
   - 只读 `data-layer/templates/` 目录（读取模板库）
   - 只写 `data-layer/templates/` 目录（保存新模板）
   - 不直接操作员工和项目数据

2. **推荐策略规则**：
   - 优先使用模板库推荐（匹配度≥80%）
   - 其次使用组合推荐（匹配度50%-80%）
   - 最后使用AI分析（匹配度<50%）
   - 所有推荐都要说明依据

3. **AI分析规则**：
   - AI推荐要基于项目特点，不能随意推荐
   - 每个岗位都要说明"为什么需要"
   - 提供替代方案和权衡说明
   - AI推荐要标注"建议根据实际情况调整"

4. **模板保存规则**：
   - AI推荐的配置要询问用户是否保存
   - 保存的模板要符合Skill 7的格式
   - 模板要标注来源（source: skill8_ai_recommendation）
   - 保存后要告知用户模板已加入知识库

5. **用户体验规则**：
   - 支持多轮对话，允许用户调整配置
   - 提供多个替代方案（预算限制、人员限制等）
   - 每个推荐都要说明依据和权衡
   - 部署过程要显示进度
