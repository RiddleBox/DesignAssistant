### 2. 领域无关性 ✨

**Skill 4：通用资产分类**
- **docs**（文档）：适用于需求文档、设计文档、API文档、用户手册等
- **code**（代码）：适用于源代码、脚本、配置代码、SQL脚本等
- **design**（设计）：适用于UI设计稿、原型、图标、插画等
- **config**（配置）：适用于环境配置、部署配置、工具配置等
- **other**（其他）：适用于任何其他类型资产

**不预设特定领域的分类**，支持用户通过标签自定义分类。

**适用场景**：
- 软件开发：代码、文档、设计稿
- 内容创作：文章、视频、图片
- 咨询项目：报告、分析文档、演示文稿
- 游戏策划：策划文档、数值表、关卡设计
- 独立游戏开发：代码、美术资源、音效、配置

**Skill 3：通用任务管理**
- **任务状态**：待办/进行中/已完成/阻塞（适用于任何项目）
- **任务优先级**：高/中/低（通用优先级）
- **任务依赖**：前置任务、阻塞关系（通用依赖模型）
- **里程碑**：目标、截止日期、关联任务（通用里程碑模型）

**不绑定特定工作流**，支持用户自定义任务状态和流程。

**适用场景**：
- 软件开发：需求分析 → 设计 → 开发 → 测试 → 发布
- 内容创作：选题 → 撰写 → 编辑 → 审核 → 发布
- 咨询项目：调研 → 分析 → 方案设计 → 汇报 → 交付
- 游戏策划：概念设计 → 系统设计 → 数值设计 → 文档编写 → 评审
- 独立游戏开发：原型 → 美术 → 程序 → 测试 → 发布
