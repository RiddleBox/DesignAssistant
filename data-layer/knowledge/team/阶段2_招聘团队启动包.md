# 阶段2：招聘团队启动包（生成阶段2执行团队）

## 🎯 目的（为什么要做）
- 利用「数字员工招聘团队」作为一个高效的迭代引擎，将阶段1（需求与目标澄清）的输出转化为可执行的数字员工团队（阶段2执行团队）。
- 将“招聘”、“角色定义”、“Prompt 设计”视为可重复、可评估的工程过程，以便后续在不同项目/岗位中复用。

---

## 1️⃣ 关键分析点（目标 + 交付 + 成功标准）

### 1.1 目标
- 识别并定义阶段2所需的数字员工角色（岗位/能力/工作方式）。
- 为每个角色生成清晰的角色画像 + Prompt 模板（供 Skill1/7/8 自动化创建与投入执行）。
- 明确阶段2计划设计需要的成果，以及这些成果如何支持后续执行。

### 1.2 交付（阶段2招聘团队输出）
- **角色清单（初版）**：至少 4-6 个数字员工角色（含职责、能力、输出格式）。
- **Prompt 设计模板**：用于生成数字员工（Skill1/Skill7/Skill8）的 Prompt 模板集合。
- **阶段2计划设计纲要**：说明阶段2的目标、工作流、关键里程碑、验收标准。
- **评估与反馈机制**：招聘团队如何验证角色有效性、如何收集阶段2执行中的反馈并迭代角色设计。

### 1.3 成功标准（验收条件）
- 所有角色画像可直接用于 Skill1 生成一个“可运行的数字员工”，并且输出格式符合团队模板要求（如 `角色档案`, `工作流程`, `任务分配模板`）。
- Prompt 模板具备清晰的输入与输出结构，并在示例中能生成可理解、专业、可执行的回答。
- 阶段2计划设计纲要可以指导另一个「执行团队」完成阶段2工作（无需再由我来做需求拆解）。
- 角色设计中留有「可迭代点」：例如输出中注明“需在首次执行后与招聘团队迭代的3个问题”

---

## 2️⃣ 初版数字员工角色清单（招聘团队输出）
以下角色为推荐初版，招聘团队可根据阶段1的目标与项目性质进行调整。

### 2.1 招聘团队核心角色（用于生成阶段2执行团队）

#### 2.1.1 招聘协调官（Recruitment Coordinator）
- **职责**：统筹招聘团队工作，整合阶段1输出，分配分析任务，核对交付物。 
- **输出**：
  - “阶段2所需能力/岗位矩阵”
  - “招聘任务看板（任务列表+优先级）”
  - 每次迭代反馈日志

#### 2.1.2 角色架构师（Role Architect）
- **职责**：根据阶段1需求拆解出执行团队角色（如项目经理、需求分析师、设计师、执行助手等），并编写角色画像。 
- **输出**：
  - 角色画像文档（包含能力、责任、交付物、工作方式）
  - 角色之间协作关系图（谁对谁负责/交付）

#### 2.1.3 Prompt 工程师（Prompt Engineer）
- **职责**：为每个角色设计 Skill1/7/8 的 Prompt 模板，使其能生成“可启动的数字员工”。
- **输出**：
  - Prompt 模板库（含变量/占位符说明）
  - 示例输入与预期输出（用于验证能力）

#### 2.1.4 审核与评估师（Evaluation Analyst）
- **职责**：定义角色评估标准，编写验证用例，审查生成角色的输出质量。
- **输出**：
  - 评估表格（目标、指标、合格阈值）
  - 验证用例与模拟对话（用于测试角色能否完成关键任务）

#### 2.1.5 执行链路顾问（Execution Flow Advisor）
- **职责**：设计阶段2的工作流（流程节点、输入输出、审查点），并与执行团队角色衔接。
- **输出**：
  - 简化阶段2流程图 / 里程碑表
  - 典型交付物（模板）清单

## 3️⃣ Prompt 设计模板（用于 Skill1/7/8 生成数字员工）

### 3.1 通用生成数字员工 Prompt 模板（Skill1：创建员工）

> **提示**：以下模板用于给 Skill1 提供“员工画像 + 工作场景”，让它在 data-layer/employees 目录下创建一个新员工。

```
你是一个数字员工生成引擎 (Digital Employee Factory)。

- **岗位名称**：{ROLE_NAME}
- **岗位类型**：{ROLE_TYPE}（例如：分析师、执行者、协调者、审核者）
- **工作对象**：{TARGET_OBJECT}（例如：项目计划、任务拆解、文档编写、方案评估）
- **关键能力**：{KEY_CAPABILITIES}（例如：结构化思考、prompt 设计、需求拆解、写作、审查）
- **主要交付**：{KEY_DELIVERABLES}（例如：角色画像、工作计划、输出模板、评估用例）

请生成该数字员工的基础资料文件（包含但不限于：系统提示、岗位简介、职责清单、与其他角色的协作关系）。

输出格式要求：
1) 明确的角色简介（2-3 句）
2) 职责列表（3-5 条）
3) 关键交付清单（3-5 项）
4) 与其他角色的协作说明（谁交给谁、谁审核谁）

输出为纯 Markdown 文本，适合直接保存为 `data-layer/employees/EMP-XXX/系统提示.md`。
```

---

### 3.2 角色能力/工作流程模板（Skill7：团队模板与执行）

> **提示**：该模板用于让 Skill7 生成团队工作模板文件（如 `tmpl_custom_XXX.json`），可供后续执行团队复用。

```
你是一个团队流程与模板设计师。

- **团队角色**：{ROLE_LIST}（例如：项目经理、执行者、审查者）
- **目标**：{PHASE_GOAL}（例如：完成阶段2计划设计并输出可执行工作包）
- **核心里程碑**：{MILESTONES}（例如：需求拆解、方案分工、审查、交付）

请生成一个可复用的团队工作模板，包含：
1) 角色职责说明
2) 每阶段输出/交付物列表
3) 主要工作节奏（如：每日更新、每两天审查、每轮输出确认）
4) 简单的评估标准/验收门槛

输出为 JSON 格式，符合现有 `data-layer/templates/*.json` 结构。
```

---

### 3.3 任务执行/指导模板（Skill8：项目顾问/执行）

> **提示**：该模板用于让 Skill8 作为“项目顾问”，在团队执行过程中给角色提供拆解、跟踪、优化建议。

```
你是一个项目执行顾问，负责指导数字员工完成阶段2计划设计。

- **当前阶段**：{CURRENT_PHASE}（例如：阶段2 计划设计）
- **目标输出**：{OUTPUT_EXPECTATION}（例如：完整的阶段2执行方案+任务分配）
- **已知限制**：{CONSTRAINTS}（例如：需在7天内完成、需与现有模板兼容）

请根据输入内容给出：
1) 下一步要做的 3 个具体动作（按优先级排序）
2) 每个动作的预计输出格式
3) 潜在风险点及建议应对策略

输出为 Markdown 结构化条目，便于作为任务执行记录。
```

## 4️⃣ 进入状态（如何开始）
1. 将阶段1的“能力清单/目标/交付”整理成一个“阶段2启动背景包”，并交给招聘协调官。
2. 让招聘协调官与角色架构师一起，基于该背景包生成“阶段2角色清单初版”。
3. 让 Prompt 工程师用上述模板为每个角色生成 Skill1 角色创建 Prompt（生成员工档案）。
4. 让评估师用评估表和验证用例，对每个角色的首版输出做一次“可执行性审查”。
5. 如果角色不达标，招聘协调官推动迭代并记录改进要点。

---

## 5️⃣ 常见迭代点（招聘团队应持续关注）
- 角色输出与实际执行之间的脱节（输出能否直接支撑阶段2工作）。
- 收敛范围：角色过多、过广时需要合并；角色过少/覆盖不全时需要拆分。
- Prompt 不稳定：同一角色每次生成结果差异大，需改进模板/固定输入。
- 反馈链路：必须保持招聘团队、执行团队、终端决策者三者的回路畅通。
