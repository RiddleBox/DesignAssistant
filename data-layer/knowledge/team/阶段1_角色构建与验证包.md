# 阶段1 角色构建 + 验证包（可直接执行）

> 本文档是「阶段1团队构建与验证」的可执行包：包括阶段1角色构建指令、Prompt 生成模板、验证用例、迭代记录。可直接交给招聘团队执行。

---

## 1️⃣ 核心目标（阶段1）
- 以阶段0输出（能力清单/交付清单/成功标准）为基础，构建阶段1数字团队
- 阶段1团队输出：
  1. 阶段1工作计划（里程碑、输出、验收标准）
  2. 阶段1交付物（计划文档、角色责任图、输出模板）
  3. 阶段2目标说明（将阶段1成果转化为阶段2输入）
- 确保阶段1输出可直接作为阶段2构建的基础，形成“阶段间递归衔接”。

---

## 2️⃣ 阶段1团队构建流程（招聘团队执行）

### 步骤1：准备输入（阶段0产出）
从现有阶段0材料中整理：
- 能力清单（AI Native / 第一性原理 / 系统化 / 协作 / 可复用）
- 成功标准（可写简历成果、可跑通workflow、可复用）
- 交付清单（岗位能力映射报告、核心架构图、评估案例、模板、复盘）

### 步骤2：角色构建（阶段1团队成员）
使用下面的“角色生成模板”生成阶段1团队的数字员工（可通过 Skill1 生成）：
- 阶段1计划设计师（Phase1 Planner）
- 阶段1协作架构师（Phase1 Collaboration Architect）
- 阶段1交付产物设计师（Phase1 Deliverable Designer）
- 阶段1验证与反馈官（Phase1 Verifier）
- 阶段1招聘协调官（Phase1 Recruitment Coordinator）

### 步骤3：阶段1角色产出验证（验证用例）
为每个角色运行一次“基线验证用例”，并根据输出执行打分与迭代。

### 步骤4：阶段1输出包装（交付阶段2输入）
输出阶段1成果包，并形成阶段2目标说明文档（阶段2目标、阶段2主要工作、关键风险/依赖）。

---

## 2.5 标准工作流位置（步骤示意）

阶段0完成 → 产出能力/交付/成功标准  
↓（输入给角色构建包）  
阶段1：角色构建与验证（使用“阶段1角色构建与验证包”）  
→ 生成阶段1角色（数字员工）  
→ 运行验证用例并迭代角色/Prompt  
↓（输出为阶段1团队+角色集、验证结论）  
阶段1执行（阶段1团队开始工作）  
→ 生成阶段1计划文档、阶段1交付、阶段2目标说明  
↓（输出为阶段2输入）  
阶段2构建（重复同样的方法）

## 3️⃣ 阶段1角色生成模板（Prompt for Skill1）

### 3.1 通用生成指令（适用于所有阶段1角色）

```
你是一个数字员工生成引擎（Digital Employee Factory）。

当前阶段：阶段1（计划设计）

请基于以下内容生成一个数字员工角色：
- 阶段0输入：{PHASE0_INPUT_SUMMARY}
- 阶段1目标与工作范围：{PHASE1_GOAL_AND_SCOPE}

请输出：
1) 角色名称（英文+中文，带阶段标签）
2) 角色简介（2-3 句）
3) 关键职责（3-5 条）
4) 关键交付（3-5 项，含输出格式/模板）
5) 与其他角色的协作关系（谁交给谁、谁审核谁）
6) 该角色的验证用例（至少 1 个），包括输入、预期输出、验收标准

输出为纯 Markdown，便于直接存入 `data-layer/employees/EMP-XXX/系统提示.md`。
```

### 3.2 角色生成示例（可直接填充）

- `PHASE0_INPUT_SUMMARY`：可包含阶段0能力清单、成功标准、交付清单的精简摘要。
- `PHASE1_GOAL_AND_SCOPE`：本文件第2节“阶段1目标与工作范围”部分。

## 4️⃣ 角色验证用例（模板）

### 4.1 验证用例结构

- **用例名称**：
- **角色**：
- **输入**：阶段0输出 + 当前阶段1目标说明
- **预期输出**：符合阶段1计划文档结构/格式的草稿
- **验收标准**：
  - 是否覆盖阶段1目标的关键点？
  - 是否给出了阶段2目标与主要工作？
  - 输出是否具备“可执行/可检查”的结构？

### 4.2 角色验证流程（推荐）
1. 使用角色生成 Prompt 运行 Skill1，生成角色“系统提示”。
2. 用该角色执行一个验证用例（可以让角色“输出阶段1计划文档或目标说明”）。
3. 评估输出是否满足验收标准，并记录得分/问题点。
4. 若未达标，返回角色生成步骤，调整角色职责或 Prompt，重复。
