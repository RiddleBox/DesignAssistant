> 3. 招聘团队输入（每个角色的 Prompt + 输出模板）

## 1 阶段计划表（阶段/任务/交付/验收）

### 1.1 五大阶段概览

| 阶段 | 目标 | 核心产出 |
|------|------|----------|
| 0. 需求澄清（已完成） | 明确能力清单 + 成功标准 + 交付清单 | 能力映射报告 + 交付清单 + 验收模板 |
| 0.5 招聘准备 | 明确需要招聘的数字员工角色与招聘方案 | 招聘需求文档 + 角色画像 + 招聘流程模板 |
| 1. 计划设计（本次输出） | 将交付拆成阶段步骤、定位所需能力与角色 | 阶段计划表 + 角色责任表 |
| 2. 团队构建（生成数字员工） | 根据计划创建专业数字员工（角色+Prompt+产出标准） | 角色画像 + Prompt 模板 + 创建流程 |
| 3. 执行交付（Workflow 运行） | 用数字员工执行计划并产出作品 | 评估案例输出 + 日志 + 复盘 |
| 4. 可复用包装（方法论产品化） | 把流程打包成可复制“产品/模板” | Workflow 模板 + 交付手册 |

---

### 1.2 阶段计划表（按阶段/任务/交付/验收）

#### 阶段 0.5：招聘准备
- **任务 0.5.1：定义数字员工招聘需求（角色+能力标签+数量）**
- **任务 0.5.2：设计招聘流程（候选生成、筛选、创建）**
- **交付**：招聘需求文档 + 角色画像 + 招聘流程模板
- **验收**：招聘团队可用该文档直接创建所需数字员工角色

#### 阶段 1：需求与目标确认
- **任务 1.1：生成能力清单 & 成功标准**
- **任务 1.2：生成交付清单（可写简历的成果）**
- **交付**：能力映射报告 + 交付清单 + 评估模板
- **验收**：文档可直接写入简历/面试成果，并能作为评估依据

#### 阶段 2：系统设计 & Workflow 规划
- **任务 2.1：设计核心工作流（多AgentCooperation/ Multi Agent Debate / 知识库 / RAG / 决策）**
- **任务 2.2：定义每个模块输入/输出/责任**
- **交付**：系统架构图 + 输入/输出规范 + 执行路径
- **验收**：任意人可根据说明复现一次评估输出

#### 阶段 2.1：招聘团队迭代（角色打磨与 Prompt 优化）
- **任务 2.1.1：用招聘团队产出初版角色画像与能力标签
  并确认这些角色是否能支撑阶段2/3的工作流程。**
- **任务 2.1.2：设计并执行“角色验证用例”，验证角色输出是否可直接用于执行交付。**
- **任务 2.1.3：根据验证结果迭代角色描述、Prompt 模板与输出规范。
  形成“角色稳定版”后才进入阶段3。**
- **交付**：可执行角色画像 + Prompt 模板 + 迭代记录（问题/改进点）
- **验收**：经过 1-2 轮迭代后，角色产出一致、可复用、能支撑阶段3执行。

> 说明：本阶段确保生成的数字员工角色足够清晰、稳定，后续阶段基于该角色输出执行内容时能保持一致性，减少反复修正成本。

#### 阶段 3：角色定义 & 团队构建
- **任务 3.1：定义数字员工角色**
- **任务 3.2：为每个角色写 Prompt + 输出模板 + 验收标准**
- **交付**：角色画像 + Prompt 模板 + 创建流程
- **验收**：角色可通过 Skill 1/7/8 创建，并能产出可检验的结果

#### 阶段 4：执行 & 产出
- **任务 4.1：按 workflow 运行一次真实评估案例**
- **任务 4.2：记录日志并完成复盘报告**
- **交付**：评估结果包 + 执行日志 + 复盘报告
- **验收**：输出可直接作为案例展示，并满足评估标准

#### 阶段 5：产品化 / 可复用
- **任务 5.1：将流程打包为可复用模板**
- **任务 5.2：生成执行指南 + 角色指南 + 复盘指南**
- **交付**：Workflow 模板 + 执行手册
- **验收**：他人按手册执行一次，产出一致结果

## 2 数字员工角色责任表（按角色+输出+验收）

### 2.1 角色：需求澄清官（Capability Architect）
- **职责**：把岗位需求转成能力清单、交付清单、成功标准。
- **输出**：
  - 能力映射文档
  - 交付清单（简历成果模板）
  - 评估模板（评估标准/打分机制）
- **验收**：文档可直接写入简历/作品集，并可用于后续评估。

### 2.2 角色：AI 工作流架构师（Workflow Architect）
- **职责**：设计多 Agent + 知识库 + 决策流水线工作流。
- **输出**：
  - 系统架构图（Agent/模块/数据流）
  - 模块分工表（输入/输出/责任）
  - 场景执行说明（谁触发、谁校验）
- **验收**：可用文档复现一次整体评估流程。

### 2.3 角色：Prompt 设计师（Prompt Engineer）
- **职责**：把系统模块拆成可运行 Prompt / Agent。
- **输出**：
  - Prompt 模板库（角色/场景/输出结构）
  - 数据格式规范（模块间传递的数据结构）
  - 结果校验模板（输出结构/质量指标）
- **验收**：Prompt 可执行并产生结构化输出。

### 2.4 角色：执行验证官（Execution Verifier）
- **职责**：运行 workflow，产出评估结果并复盘。
- **输出**：
  - 评估案例产出包（输入+输出+结论）
  - 执行日志（包含步骤、时间、问题）
  - 复盘报告（流程有效性、改进建议）
- **验收**：输出满足成功标准，可作为展示案例。

### 2.5 角色：招聘协调官（Recruitment Coordinator）
- **职责**：负责根据阶段0/阶段1计划，组建所需数字员工团队（招聘+角色创建）。
- **输出**：
  - 数字员工招聘需求文档（角色画像 + 能力标签 + 数量）
  - 招聘流程模板（候选生成、筛选、创建、验证）
  - 招聘完成报告（已创建数字员工列表 + 角色说明）
- **验收**：根据文档可直接执行创建所需数字员工，创建结果满足阶段计划所需角色定义。

## 3 招聘团队的输入（每个角色需要哪些 Prompt + 输出模板）

### 3.1 角色：需求澄清官（Capability Architect）

**目标 Prompt**：
> “根据以下岗位描述，生成一个能力清单、交付清单、以及对应的评估/验证标准。
> 输出格式：
> 1. 能力清单（每项包含定义+目标）
> 2. 交付清单（每项包含说明+输出形式）
> 3. 验证模板（每条能力对应的评估方法）”

**输出模板**：
- 能力清单表：能力 | 目标 | 评估指标
- 交付清单表：交付物 | 格式 | 交付标准
- 评估模板：评估项 | 验证方式 | 合格线

---

### 3.2 角色：AI 工作流架构师（Workflow Architect）

**目标 Prompt**：
> “根据以下能力清单和交付清单，设计一个多 Agent + 知识库 + RAG 工作流系统。
> 要求输出：系统架构图概要、模块职责说明、数据流说明、触发逻辑说明。”

**输出模板**：
- 系统架构图（文本描述+关键组件列表）
- 模块职责表：模块 | 角色 | 输入 | 输出 | 关键指标
- 数据流说明：数据来源 → 处理 → 输出路径

---

### 3.3 角色：Prompt 设计师（Prompt Engineer）

**目标 Prompt**：
> “对于每个系统组件，生成可直接执行的 Prompt/Agent 配置，包含：
> 1) 角色身份与职责
> 2) 输入要求与输出格式
> 3) 验证规则（如何知道输出正确）”

**输出模板**：
- Prompt 模板：角色 | 目的 | 输入结构 | 输出结构
- 校验模板：期望输出示例 + 验证逻辑

---

### 3.4 角色：执行验证官（Execution Verifier）

**目标 Prompt**：
> “使用上述 workflow（包含 Agent/Prompt/知识库）执行一次真实评估，记录全过程。
> 需要输出：
> - 评估输入（问题/场景）
> - 评估输出（结论/建议/行动）
> - 执行日志（每一步的输入/输出/问题）
> - 复盘报告（流程成功点/问题点/改进建议）”

**输出模板**：
- 评估报告：输入 | 结果 | 建议
- 执行日志：步骤 | 输入 | 输出 | 结果 | 问题
- 复盘报告：亮点 | 问题 | 改进
