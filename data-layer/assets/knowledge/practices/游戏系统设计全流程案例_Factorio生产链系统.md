# 游戏系统设计全流程案例：Factorio（异星工厂）生产链系统（叙事重构版）

## 📋 案例概述

本文以《Factorio》的“生产链系统”为核心，按《星露谷物语农场系统》样本的研究写法，重构为**“玩家旅程/里程碑驱动”**的叙述：先把玩家为什么非做不可讲清楚（动机闭环与节奏引擎），再在每个阶段自然引出生产、物流、科技、电力与压力系统。

- **设计时间线**：2012-2020（Early Access 2016-2020）
- **核心体验承诺**：从手工劳动到完全自动化的进化；“看到物品在系统中流动”的成就感；“永远能更好”的优化空间。

**对照星露谷（为了读法一致）**
- 星露谷把节奏建立在**时间预算/季节变化**上：每天做选择、每季换目标、冬季强迫转向。
- Factorio 把节奏建立在**吞吐量/枯竭/外部压力**上：解决瓶颈→扩张→暴露新瓶颈，且研究/资源/压力三条循环互相加速。

---

## 🎯 第一阶段：愿景、核心循环与“动机引擎”

### Day 1：确定愿景（设计师内心独白）

> “我要做一款让玩家感受自动化成就感的游戏。玩家从手挖矿开始，逐步建造机器，最终拥有自运行的巨型工厂。看着传送带上物品流动，本身就是奖励。”

```
【设计文档 v0.1】
- 类型：工厂建设 + 自动化 + 生存压力
- 核心体验：从手工到自动化的进化
- 目标玩家：喜欢规划、优化、问题解决
- 关键问题：
  1) 玩家每小时在解决什么问题？
  2) 自动化之后玩家还要做什么？
  3) 如何让系统越来越复杂但仍可理解？
```

### Day 2-3：核心循环（不是一条，而是三条互相咬合）

在“动作层”上，玩家的循环很简单：

```
发现瓶颈 → 扩产/换方案 → 产能上升 → 新瓶颈暴露 →（循环）
```

但让玩家**必须持续推进**的，是三条“动机闭环”（节奏引擎）：

- **研究闭环（进度引擎）**：科技包持续吞吐 → 解锁新能力 → 新能力抬高基础材料需求 → 需要扩产与优化 → 科技包吞吐继续提高
- **资源闭环（扩张引擎）**：矿点衰减/枯竭 → 远距离开采 → 物流升级（火车/更高吞吐）→ 产能变大 → 消耗更快 → 继续扩张
- **压力闭环（对抗引擎）**：产能扩大 → 污染增加 → 敌人压力上升 → 防御/军工需求上升 → 产能进一步扩大

**👁️ 旁观者观察：**
> 这就是“为什么要做生产线/为什么要做新科技”的根：研究提供方向、枯竭提供迁移压力、污染提供外部代价，三者让玩家的自动化永远“有下一步”。

## 🧭 第二阶段：玩家旅程与里程碑（动机与模块的衔接点）

像星露谷用“日/周/季/年”组织玩家体验一样，Factorio 也可以用“阶段里程碑”组织：每个阶段都有**明确目标**、**强迫学习点**与**典型卡点**，模块也在此处自然登场。

### 阶段总览

| 阶段 | 玩家目标（为什么要做） | 关键里程碑（做成什么算过关） | 被迫学习点（学会什么） |
|---|---|---|---|
| 手工时代 | 把“手工痛苦”变成动机 | 第一台采矿机 + 第一段传送带 | 自动化的价值、最小供给链 |
| 半自动时代 | 建造速度/需求增长逼你扩产 | 第一条稳定的齿轮/板材产线 | 供需、瓶颈、节拍 |
| 自动化时代 | 研究吞吐变成长期债务 | 红瓶→绿瓶科研持续运行 | 多输入配方、分流、协同 |
| 规模化时代 | 枯竭与吞吐冲突 | 火车解锁并承担资源干线 | 模块化、站点化、网络化 |
| 对抗时代（贯穿） | 污染把扩张变成有代价 | 前线防线 + 自动补给 | 外部成本、军工产线 |
| 终局 | “系统验收”与完成感 | 发射火箭 | 全链路稳定性与冗余 |

## 🔧 第三阶段：核心系统拆解（对照星露谷的“系统边界/接口”）

为了让复杂系统可控，像星露谷样本那样先明确系统职责与边界（做什么、不做什么）。

- **生产系统（ProductionSystem）负责**：配方、机器、产出速率、并行化；**不负责**：运输路径选择、电力供给、敌人行为。
- **物流系统（LogisticsSystem）负责**：传送带/插臂/火车/机器人等运输能力与约束；**不负责**：配方逻辑与科研解锁。
- **科技系统（ResearchSystem）负责**：科技树、研究吞吐、解锁项；**不负责**：替玩家“自动规划”生产线（把问题留给玩家）。
- **电力系统（PowerSystem）负责**：供需平衡、低电降速、发电扩容；**不负责**：直接产出物品。
- **压力系统（PressureSystem：污染/敌人）负责**：扩张代价与威胁曲线；**不负责**：替玩家修复或优化布局。

示意接口（研究视角）：
- `ResearchSystem.consume(sciencePackType, rate)` → 持续消耗
- `ResearchSystem.unlock(techId)` → 解锁新配方/建筑
- `ProductionSystem.produce(item, rate)` → 目标产出速率
- `LogisticsSystem.transport(item, from, to, throughput)` → 运力约束
- `PowerSystem.brownoutFactor()` → 低电反馈
- `PressureSystem.threatLevel()` → 外部压力反馈

**👁️ 旁观者观察：**
> 这套边界决定了“自洽”：研究与生产绑定，但研究不替玩家解决布局；物流提供工具，但不消灭瓶颈；压力系统制造代价，但不强制失败。

## 🏭 第四阶段：生产链系统（从里程碑出发的设计）

### 4.1 为什么是“铁齿轮”？（第一道动机闸门）

“玩家为什么要做齿轮？”不是因为真实，而是因为它在系统层面承担了**早期通用中间件**的角色。

- **低门槛、高复用**：配方简单、产出快，但需求覆盖面极广（传送带、机械臂、组装机、科研等）。
- **跨系统耦合**：用一个材料把“生产/物流/科技”的升级扣成一条线。

```
铁矿 → 铁板 → 齿轮 →（传送带 / 机械臂 / 组装机 / 科技包 …）
```

- **正反馈极强**：齿轮自动化后，基地建造速度立刻上升——这就是“自动化的甜头”。

**👁️ 旁观者观察：**
> 星露谷里“洒水器”解决的是重复劳作与时间预算；Factorio 里“齿轮产线”解决的是重复合成与建设速度。两者都是早期“解放劳动”的关键节点。

### 4.2 层级设计：为什么要“这么多层级”？（先给答案，再给例子）

你指出的点非常关键：如果只说“认知负载/节奏/投资/瓶颈”但不说明它们**共同在解决什么问题**，读起来会像口号。

**先回答：为什么要设计这么多层级？**
- **让玩家的学习与工厂的增长同步**：玩家能力是阶梯式增长的（从“会做”到“能稳定做”再到“规模化做”），层级就是把学习拆成若干次可消化的跃迁。
- **让进度变成“系统能力验收”而不是“交作业”**：每个新层级都要求你证明自己已经具备某种工业能力（多输入协同、连续供应、网络化物流、全局供电等），否则就会在瓶颈处自然停下。
- **把“扩张”塑造成连续的重构游戏**：每次新层级都不是“加一个新配方”，而是把旧体系的假设推翻（吞吐不够、供电不稳、资源枯竭、污染压力变大），迫使玩家进行一次“工程重构”。

---

## 📊 总结：可复用的设计原则

1. **用里程碑组织复杂度**：阶段目标明确，玩家知道“下一步要学什么”。
2. **把进度做成持续吞吐**：研究不是交付，而是产能证明与长期债务。
3. **故意留出不完美比例**：让优化成为玩法，而不是让系统自动完美。
4. **让问题可见、可算、可解**：物品流动可见、统计可视化辅助决策。
5. **扩张必须有代价**：枯竭与污染让“更大”不是免费的，才会出现策略选择。
