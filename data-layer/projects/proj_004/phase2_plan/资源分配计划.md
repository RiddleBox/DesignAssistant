## 3. 预算分配

### 3.1 总体预算

| 类别 | 明细 | 成本 | 占比 |
|------|------|------|------|
| LLM API | Claude Opus/Sonnet | $650 | 65% |
| Embedding API | text-embedding-3-large | $65 | 6.5% |
| 向量数据库 | Qdrant（自托管） | $0 | 0% |
| 服务器 | 开发/测试环境 | $200 | 20% |
| 其他工具 | 订阅/许可 | $85 | 8.5% |
| **总计** | - | **$1000** | **100%** |

### 3.2 分阶段预算

| 子阶段 | 预算 | 主要支出 |
|--------|------|---------|
| 2.0 规划 | $0 | 无API调用 |
| 2.1 情报解码 | $150 | LLM API |
| 2.2 机会评估 | $250 | LLM API（多Agent） |
| 2.3 决策建议 | $120 | LLM API |
| 2.4 知识库RAG | $95 | Embedding API |
| 2.5 整合验证 | $100 | LLM API |
| 基础设施 | $285 | 服务器、工具 |
| **总计** | **$1000** | - |

### 3.3 预算控制措施

1. **成本监控**：
   - 每日统计API调用量和成本
   - 每周汇报预算使用情况
   - 超过80%预警，超过90%限制

2. **降本措施**：
   - 优先使用Sonnet 4.6（成本更低）
   - 优化Prompt，减少Token消耗
   - 使用缓存，避免重复调用

3. **应急预算**：
   - 预留$200应急预算（20%）
   - 用于处理超预期情况

## 5. 资源冲突与解决

### 5.1 人力资源冲突

**冲突1：Week 3-4人力峰值（8人）**
- **冲突**：2.1/2.2/2.4团队同时运行
- **解决**：
  - 2.1团队Week 3末完成，释放3人
  - 2.2团队Week 3开始，需要4人
  - 实际峰值：Week 3前半周（7人），Week 3后半周（8人）

**冲突2：测试工程师需求重叠**
- **冲突**：2.1和2.5都需要测试工程师
- **解决**：
  - EMP-023在2.1完成后（Week 3末）加入2.5
  - 2.5另招1名测试工程师（EMP-036）

### 5.2 技术资源冲突

**冲突1：LLM API调用量峰值**
- **冲突**：Week 3-5多个团队同时调用
- **解决**：
  - 设置每日调用上限（100次Opus）
  - 优先级排序（2.5 > 2.2 > 2.1 > 2.3）
  - 使用Sonnet降级

**冲突2：向量数据库访问**
- **冲突**：2.4构建索引时，其他团队需要查询
- **解决**：
  - 2.4优先构建基础索引
  - 增量更新不影响查询
  - 读写分离

## 6. 资源优化建议

### 6.1 人力优化

1. **角色复用**：测试工程师、案例工程师可跨团队复用
2. **弹性配置**：非关键路径的角色可以兼职
3. **技能培训**：提前培训，提高人员利用率

### 6.2 成本优化

1. **模型选择**：非关键任务使用Sonnet 4.6
2. **Prompt优化**：减少Token消耗
3. **缓存机制**：避免重复调用

### 6.3 时间优化

1. **并行执行**：最大化并行，缩短总时间
2. **提前准备**：数据、工具提前准备好
3. **快速迭代**：小步快跑，及时调整

## 7. 资源监控与调整

### 7.1 监控指标

| 指标 | 目标 | 监控频率 | 负责人 |
|------|------|---------|--------|
| 人力利用率 | ≥80% | 每周 | EMP-020 |
| API成本 | ≤$1000 | 每日 | EMP-020 |
| 进度达成率 | ≥90% | 每周 | EMP-020 |
| 资源冲突次数 | ≤3次/周 | 每周 | EMP-020 |

### 7.2 调整机制

**触发条件**：
- 人力利用率<60%或>95%
- API成本超过预算80%
- 进度延期>3天
- 资源冲突频繁（>5次/周）

**调整流程**：
1. 识别问题根因
2. 提出调整方案
3. 与相关团队沟通
4. 执行调整
5. 跟踪效果
