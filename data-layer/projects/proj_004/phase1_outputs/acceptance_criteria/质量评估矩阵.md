## 评估维度

### 1. 完整性（Completeness）

**定义**：输出物是否包含所有必要的内容和信息。

| 评分 | 标准 | 说明 |
|------|------|------|
| 5分 | 完全完整 | 包含所有必要内容，无遗漏 |
| 4分 | 基本完整 | 包含核心内容，缺少少量次要信息 |
| 3分 | 部分完整 | 包含主要内容，但有明显遗漏 |
| 2分 | 不够完整 | 缺少重要内容，影响使用 |
| 1分 | 严重不完整 | 缺少核心内容，无法使用 |

**检查要点**：
- [ ] 是否包含所有必需章节/模块？
- [ ] 是否回答了所有关键问题？
- [ ] 是否提供了必要的示例/说明？
- [ ] 是否定义了所有关键术语？

---

### 2. 准确性（Accuracy）

**定义**：输出物的内容是否准确、正确、无误。

| 评分 | 标准 | 说明 |
|------|------|------|
| 5分 | 完全准确 | 所有信息准确无误 |
| 4分 | 基本准确 | 核心信息准确，有少量小错误 |
| 3分 | 部分准确 | 主要信息准确，但有明显错误 |
| 2分 | 不够准确 | 存在重要错误，影响可信度 |
| 1分 | 严重错误 | 核心信息错误，不可使用 |

**检查要点**：
- [ ] 数据/数字是否准确？
- [ ] 逻辑推理是否正确？
- [ ] 引用/参考是否准确？
- [ ] 技术描述是否正确？

---

### 3. 一致性（Consistency）

**定义**：输出物内部及与其他输出物之间是否保持一致。

| 评分 | 标准 | 说明 |
|------|------|------|
| 5分 | 完全一致 | 内部一致，与其他文档完全对齐 |
| 4分 | 基本一致 | 核心内容一致，有少量不一致 |
| 3分 | 部分一致 | 主要内容一致，但有明显冲突 |
| 2分 | 不够一致 | 存在重要冲突，影响理解 |
| 1分 | 严重冲突 | 核心内容冲突，无法使用 |

**检查要点**：
- [ ] 术语使用是否一致？
- [ ] 格式风格是否统一？
- [ ] 与系统架构是否对齐？
- [ ] 与阶段0目标是否一致？

---

### 4. 可用性（Usability）

**定义**：输出物是否可以直接使用，无需大幅修改。

| 评分 | 标准 | 说明 |
|------|------|------|
| 5分 | 可直接使用 | 无需任何修改即可使用 |
| 4分 | 需小幅调整 | 需要少量调整（<10%工作量） |
| 3分 | 需中度调整 | 需要一定调整（10-30%工作量） |
| 2分 | 需大幅调整 | 需要大量调整（30-50%工作量） |
| 1分 | 无法使用 | 需要重做（>50%工作量） |

**检查要点**：
- [ ] 格式是否规范？
- [ ] 说明是否清晰？
- [ ] 示例是否充分？
- [ ] 是否易于理解和执行？

---

### 5. 可复用性（Reusability）

**定义**：输出物是否可以应用于其他类似项目或场景。

| 评分 | 标准 | 说明 |
|------|------|------|
| 5分 | 高度可复用 | 可直接用于其他项目，通用性强 |
| 4分 | 较好复用 | 可用于类似项目，需少量调整 |
| 3分 | 部分可复用 | 可复用部分内容，需较多调整 |
| 2分 | 难以复用 | 仅适用于当前项目，难以迁移 |
| 1分 | 无法复用 | 完全定制化，无复用价值 |

**检查要点**：
- [ ] 是否抽象了通用模式？
- [ ] 是否避免了过度定制？
- [ ] 是否提供了可配置参数？
- [ ] 是否有清晰的使用说明？

---

## 综合评分

### 评分公式

```
综合得分 = (完整性 × 30% + 准确性 × 25% + 一致性 × 20% + 可用性 × 15% + 可复用性 × 10%)
```

### 评级标准

| 综合得分 | 评级 | 说明 | 处理方式 |
|---------|------|------|---------|
| 4.5-5.0 | 优秀 | 质量优秀，超出预期 | ✅ 直接通过 |
| 4.0-4.4 | 良好 | 质量良好，符合预期 | ✅ 通过 |
| 3.5-3.9 | 合格 | 质量合格，基本达标 | ⚠️ 有条件通过，建议改进 |
| 3.0-3.4 | 基本合格 | 质量基本合格，需改进 | ⚠️ 有条件通过，必须改进 |
| <3.0 | 不合格 | 质量不达标 | ❌ 不通过，需返工 |

---

## 评估流程

### 1. 单项评估
对每个维度进行独立评分（1-5分）。

### 2. 计算综合得分
使用评分公式计算综合得分。

### 3. 确定评级
根据综合得分确定评级。

### 4. 记录问题
记录所有发现的问题和改进建议。

### 5. 跟踪改进
跟踪问题的改进进度，直到达标。

### 评估原则
1. **客观公正**：基于事实和标准，不带主观偏见
2. **建设性**：提出具体可行的改进建议
3. **一致性**：对所有输出物使用相同标准
4. **透明性**：评估过程和结果公开透明
