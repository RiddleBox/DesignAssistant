## 一、执行摘要

本次 benchmark 是 Phase 2.1 情报解码模块的首轮 baseline 验证，目标是：

1. 验证当前 Prompt-first MVP 的基础可行性
2. 建立性能 baseline
3. 识别主要错误模式
4. 判断是否需要轻量优化

**核心结论**：

> **当前 MVP 在工程上可运行，Schema 输出稳定，召回率良好（83.33%），但存在明显的误报问题（Precision 仅 35.71%），建议进行一轮轻量 Prompt 和 few-shot 优化。**

## 五、主要问题诊断

### 5.1 核心问题

1. **market 信号误报严重**（25 个误报，占总误报 55.6%）
   - 泛趋势表述被误报为正式信号
   - 背景信息被升级为独立 market 信号
   - 缺少"什么不该抽"的约束

2. **边界样本过度自信**（5 个高置信度误报）
   - 模糊证据样本输出 confidence: 10
   - 推断链过长的内容未降低置信度
   - 缺少"证据边界"和"克制原则"

3. **信号类型边界模糊**
   - capital vs market 混淆（如大型收购）
   - 背景信息 vs 独立信号混淆

### 5.2 次要问题

1. **部分 team 信号漏报**（2 个漏报）
   - 可能因表述不够明确
   - few-shot 样例覆盖不足

2. **标签命名风格不统一**
   - 系统输出中文描述性标签
   - 人工标注使用英文代码式标签
   - 影响下游可比较性

## 六、优化建议

基于错误画像，建议进行**一轮轻量 Prompt 和 few-shot 优化**（2-3 项调整）：

### 6.1 P0 优化（必须）

#### 建议 1: 补充 market 类型的边界样例和负例

**目标**: 压制 market 信号误报

**具体动作**:
- 在 few-shot 中补充 1-2 个 market 负例
  - 示例：泛趋势表述 → 不应抽成正式信号
  - 示例：背景信息 → 应保留为背景，而非独立信号
- 在 few-shot 中补充 1 个 market 边界正例
  - 示例：模糊市场变化 → 应降低 confidence_score

#### 建议 2: 在角色设定中强化"证据边界"和"克制原则"

**目标**: 避免过度自信和过度推断

**具体动作**:
- 在 SYSTEM_PROMPT 中显式补充：
  ```
  重要原则：
  - 只有原文明确支持的变化才能进入正式信号
  - 推断链过长或证据模糊时，应降低 confidence_score
  - 背景信息不应升级为独立信号
  - 不确定时保持克制，允许不抽或低置信输出
  ```

### 6.2 P1 优化（建议）

#### 建议 3: 在 Prompt 中显式说明"模糊样本应降低 confidence_score"

**目标**: 提升边界样本稳定性

**具体动作**:
- 在评分口径中补充：
  ```
  confidence_score 特别说明：
  - 传闻、二手引用、推断性内容：1-3 分
  - 间接证据、表述模糊：4-6 分
  - 官方确认、直接证据：7-10 分
  ```

## 七、不建议在当前阶段做的事

为保持 MVP 迭代节奏，以下动作**不建议**在当前阶段进行：

1. **大规模重写 Prompt 体系**
   - 当前 Prompt 结构基本可用，只需轻量调整

2. **引入复杂的动态 few-shot 调度**
   - 当前静态 few-shot 已基本覆盖四类信号
   - 动态调度收益不明确，会增加复杂度

3. **立即接入 Phase 2.4 增强上下文**
   - 当前问题主要在 Prompt 角色设定和 few-shot 边界
   - 应先解决基础问题，再考虑外部增强

4. **修改 Schema 或评分维度**
   - 当前 Schema 输出稳定，问题不在结构层

5. **扩大 benchmark 样本规模**
   - 30 个样本已足够暴露主要问题
   - 应先优化，再扩大验证
