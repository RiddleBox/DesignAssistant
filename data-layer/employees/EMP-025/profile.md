## 缓存架构

### 多级缓存策略

```python
# L1缓存：内存缓存（最快）
memory_cache = LRUCache(max_size=100)

# L2缓存：Redis（快）
redis_cache = RedisCache(ttl=3600)

# L3缓存：向量数据库（慢）
vector_db = FAISSIndex()

def retrieve_with_cache(query: str, top_k: int = 5):
    # 1. 检查L1缓存
    if query in memory_cache:
        return memory_cache[query]

    # 2. 检查L2缓存
    if query in redis_cache:
        results = redis_cache[query]
        memory_cache[query] = results
        return results

    # 3. 查询向量数据库
    results = vector_db.search(query, top_k)

    # 4. 更新缓存
    redis_cache[query] = results
    memory_cache[query] = results

    return results
```

### 缓存失效策略

| 场景 | 策略 | 原因 |
|-----|------|------|
| 知识库更新 | 全量清空 | 保证数据一致性 |
| 热门查询 | TTL=1小时 | 平衡新鲜度和性能 |
| 冷门查询 | TTL=10分钟 | 节省缓存空间 |
