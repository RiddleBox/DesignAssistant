# Skill 6：团队知识库共享

## 📋 技能概述

**技能名称**：团队知识库共享  
**技能编号**：SKILL-006  
**版本**：v1.0.0  
**依赖**：Skill 1（数字员工招募与管理）、Skill 2（团队项目协作）  
**数据域**：`data-layer/knowledge/`（读写）、`projects/*/knowledge/`（读写）

**核心功能**：
1. 知识库存储和分类（个人/团队知识）
2. 知识检索和智能推荐
3. 相似知识检测和去重
4. 知识地图可视化
5. 知识引用计数和热度统计

**设计原则**：
- ✅ **领域无关**：通用知识分类，适用于任何领域
- ✅ **低耦合**：只写knowledge/目录，通过接口读取其他数据
- ✅ **零依赖**：使用系统内置命令和AI工具
- ✅ **智能推荐**：主动检索相关知识，避免重复劳动

---

## 🎯 使用场景

### 场景1：保存个人知识
```
用户："将这个技术方案保存到知识库"
→ 询问知识类型和可见范围（个人/团队）
→ 自动分类和生成标签
→ 保存到对应知识库
→ 检测相似知识并提示
```

### 场景2：检索团队知识
```
用户："查找关于API设计的知识"
→ 在团队知识库中搜索
→ 按相关度排序展示结果
→ 显示知识来源和引用次数
```

### 场景3：智能推荐
```
用户："我需要设计一个用户认证系统"
→ 自动检索相关知识
→ 推荐团队已有的认证方案
→ 避免重复设计
```

### 场景4：知识去重
```
用户："保存这个数据库优化方案"
→ 检测到相似知识已存在
→ 提示："发现相似知识，是否合并或关联？"
→ 用户选择合并/关联/独立保存
```

## 🔧 核心功能实现

### Step 0：环境检测与自动初始化

在开始任何操作前，必须先执行环境检测：

1. **检查知识库数据层**：使用 `read_file` 检查 `data-layer/knowledge/index.json` 是否存在
2. **数据层不存在** → 执行零依赖自举初始化：
   - 告知用户：\"检测到知识库数据层尚未初始化，正在自动创建...\"
   - 使用 `terminal` 创建 `data-layer/knowledge/` 目录
   - 使用 `terminal` 创建分类子目录：`team/技术方案/`、`team/业务规范/`、`team/行业趋势/`、`team/最佳实践/`
   - 使用 `edit_file` 创建 `index.json` 文件（初始化为空知识列表）
   - 使用 `edit_file` 创建 `categories.json` 文件（预定义4个分类）
   - 使用 `edit_file` 创建 `knowledge_map.json` 文件（初始化为空地图）

3. **检查项目知识库配置**：
   - 如果用户操作涉及特定项目，检查该项目的 `knowledge/` 目录是否存在
   - 不存在 → 自动创建项目知识库目录

### Step 1：意图识别

分析用户输入，识别操作意图：

**1A. 保存知识**
- 关键词：保存、记录、添加 + 知识/方案/经验
- 示例：\"将这个技术方案保存到知识库\"、\"记录这个经验\"

**1B. 检索知识**
- 关键词：查找、搜索、检索 + 知识/方案
- 示例：\"查找关于API设计的知识\"、\"搜索性能优化方案\"

**1C. 查看知识详情**
- 关键词：查看、显示 + 具体知识标题 + 详情
- 示例：\"查看'RESTful API设计最佳实践'详情\"

**1D. 查看知识库列表**
- 关键词：查看、显示、列出 + 知识库/知识列表
- 示例：\"查看团队知识库\"、\"显示所有知识\"

**1E. 更新知识**
- 关键词：更新、修改、编辑 + 知识标题
- 示例：\"更新API设计方案\"、\"修改这个知识\"

**1F. 删除知识**
- 关键词：删除、移除 + 知识标题
- 示例：\"删除这个知识\"、\"移除过时的方案\"

**1G. 查看知识地图**
- 关键词：查看、显示 + 知识地图/知识关系
- 示例：\"查看知识地图\"、\"显示知识关系\"

**1H. 知识统计**
- 关键词：统计、报告、分析 + 知识库
- 示例：\"统计知识库\"、\"生成知识报告\"

### Step 2A：保存知识

#### 阶段一：收集知识信息

1. **检查用户是否提供了知识内容**
   - 已提供（附件、代码块、文本） → 继续
   - 未提供 → 询问：\"请提供要保存的知识内容\"

2. **询问知识基本信息**：
   - \"请提供知识标题（例如：RESTful API设计最佳实践）\"
   - \"请选择知识分类（技术方案/业务规范/行业趋势/最佳实践）\"
   - \"请选择可见范围（个人/团队，默认：团队）\"
   - \"请提供知识摘要（可选，1-2句话概括）\"
   - \"请添加标签（多个标签用逗号分隔，可选）\"

3. **检查用户是否指定了项目**
   - 已指定 → 继续
   - 未指定 → 询问：\"请选择关联的项目（可选）\"

#### 阶段二：相似知识检测

1. **提取关键词**：
   - 从标题和内容中提取关键词
   - 使用标签作为额外关键词

2. **搜索相似知识**：
   - 在知识库索引中搜索包含相同关键词的知识
   - 计算相似度（关键词重叠度）

3. **如果发现相似知识**：
   ```
   ⚠️ 发现相似知识
   
   您要保存的知识：RESTful API设计最佳实践
   
   相似的已有知识：

      - 标签：API、版本管理、后端开发
   
   建议操作：
   1. 关联到已有知识（建立知识关系）
   2. 合并到已有知识（更新已有知识）
   3. 独立保存（作为新知识）
   
   请选择操作（1/2/3）：
   ```

4. **根据用户选择执行对应操作**

#### 阶段三：自动分类和生成元数据

1. **确定知识分类**：
   - 根据用户选择的分类，映射到对应的category
   - 技术方案 → 技术方案/
   - 业务规范 → 业务规范/
   - 行业趋势 → 行业趋势/
   - 最佳实践 → 最佳实践/

2. **生成知识ID**：
   - 格式：`know_YYYYMMDD_NNN`
   - 示例：`know_20260212_001`

3. **生成元数据**：
   - 作者：从当前上下文获取（如果有员工身份）
   - 创建时间：当前时间
   - 标签：用户提供的标签 + 自动生成的标签（基于分类和内容）
   - 关键词：从标题和内容中提取
   - 摘要：用户提供或自动生成（取内容前100字）

4. **生成知识文件名**：
   - 格式：`{标题}.md`
   - 示例：`RESTful_API设计最佳实践.md`

#### 阶段四：保存知识文件

1. **确定保存路径**：
   - **团队知识**：`data-layer/knowledge/team/{分类目录}/{文件名}`
   - **个人知识**：`employees/{员工ID}/知识库/{分类目录}/{文件名}`

2. **创建知识文件**：
   - 使用 `edit_file` 创建知识文件
   - 写入用户提供的内容
   - 添加元数据头部（YAML格式）

3. **知识文件格式示例**：

### Step 2B：检索知识

#### 阶段一：收集检索条件

1. **询问检索条件**：
   - \"请输入搜索关键词\"
   - \"请选择知识分类（技术方案/业务规范/行业趋势/最佳实践/全部，默认：全部）\"
   - \"请选择可见范围（个人/团队/全部，默认：全部）\"
   - \"请选择作者（可选）\"

#### 阶段二：执行搜索

1. **读取知识库索引**：使用 `read_file` 读取 `data-layer/knowledge/index.json`

2. **多维度筛选**：
   - 关键词匹配：在标题、摘要、标签、关键词中搜索
   - 分类匹配：按category筛选
   - 范围匹配：按scope筛选
   - 作者匹配：按authorId或authorName筛选

3. **计算相关度**：
   - 标题完全匹配：权重 1.0
   - 标题部分匹配：权重 0.8
   - 标签匹配：权重 0.6
   - 摘要匹配：权重 0.4
   - 关键词匹配：权重 0.3

4. **排序结果**：
   - 按相关度排序（相关度高的在前）
   - 相关度相同时，按引用次数排序
   - 引用次数相同时，按创建时间排序（最新的在前）

## 📋 工作准则

1. **数据访问规则**：
   - 只写 `data-layer/knowledge/` 和 `projects/*/knowledge/` 目录
   - 只读其他数据层目录
   - 通过接口调用其他Skill

2. **知识分类原则**：
   - 使用通用分类（技术方案/业务规范/行业趋势/最佳实践）
   - 不预设特定领域的分类
   - 支持用户通过标签自定义分类

3. **相似知识检测规则**：
   - 关键词重叠度 > 60% 视为相似
   - 提示用户选择关联/合并/独立保存
   - 不强制合并，尊重用户选择

4. **知识推荐规则**：
   - 优先推荐团队知识
   - 按相关度和引用次数排序
   - 最多推荐5条知识

5. **错误处理**：
   - 知识不存在时，展示知识列表
   - 保存失败时，提示用户并回滚
   - 删除有引用的知识时，必须警告

6. **性能优化**：
   - 知识列表分页显示（每页20条）
   - 搜索结果限制数量（最多100条）
   - 知识地图节点限制数量（最多50个）
