# Skill 2：团队项目协作

## 📋 技能概述

**技能名称**：团队项目协作管理  
**技能编号**：SKILL-002  
**版本**：v1.0.0  
**依赖**：Skill 1（数字员工招募与管理）  
**数据域**：`data-layer/projects/`（读写）、`data-layer/employees/`（只读）

**核心功能**：
1. 创建和管理项目工作空间
2. 分配员工到项目并管理权限
3. 展示项目详情和成员工作进度
4. 管理项目共享资源和配置

**设计原则**：
- ✅ **领域无关**：不绑定特定行业，适用于任何类型的项目
- ✅ **低耦合**：通过标准接口与Skill 1交互
- ✅ **零依赖**：使用系统内置命令和AI工具，无需外部依赖

## 🔧 核心功能实现

### Step 0：环境检测与自动初始化

在开始任何操作前，必须先执行环境检测：

1. **检查数据层**：使用 `read_file` 检查 `data-layer/projects/index.json` 是否存在
2. **数据层不存在** → 执行零依赖自举初始化：
   - 告知用户："检测到项目数据层尚未初始化，正在自动创建..."
   - 使用 `terminal` 创建 `data-layer/projects/` 目录
   - 使用 `edit_file` 创建 `index.json` 文件（初始化为空项目列表）
3. **检查项目工作空间根路径**：使用 `list_dir` 检查 `~/Documents/digital_projects/` 是否存在
4. **路径不存在** → 询问用户是否创建，确认后使用 `terminal` 的 `mkdir` 命令创建

### Step 2A：创建项目

#### 阶段一：收集项目信息

1. **检查用户是否提供了项目名称**
   - 已提供 → 继续
   - 未提供 → 询问："请提供项目名称"

2. **询问项目描述**（可选）
   - "请简要描述项目目标和内容（可选，直接回车跳过）"

3. **询问项目标签**（可选）
   - "请提供项目标签，多个标签用逗号分隔（可选）"

#### 阶段二：创建项目工作空间

1. **生成项目ID**
   - 格式：`proj_YYYYMMDD_NNN`
   - 示例：`proj_20260212_001`

2. **创建项目目录结构**（使用 `terminal` 工具）

   **Windows (PowerShell)**：
   ```powershell
   $projectPath = "~/Documents/digital_projects/proj_20260212_001"
   New-Item -ItemType Directory -Force -Path "$projectPath/shared/documents"
   New-Item -ItemType Directory -Force -Path "$projectPath/shared/code"
   New-Item -ItemType Directory -Force -Path "$projectPath/shared/design"
   New-Item -ItemType Directory -Force -Path "$projectPath/shared/config"
   New-Item -ItemType Directory -Force -Path "$projectPath/members"
   New-Item -ItemType Directory -Force -Path "$projectPath/config"
   ```

   **Linux/Mac (Bash)**：
   ```bash
   mkdir -p ~/Documents/digital_projects/proj_20260212_001/shared/{documents,code,design,config}
   mkdir -p ~/Documents/digital_projects/proj_20260212_001/members
   mkdir -p ~/Documents/digital_projects/proj_20260212_001/config
   ```

3. **创建项目配置文件**（使用 `edit_file` 工具）
   - 创建 `config/project.json`（参考数据结构设计）
   - 初始成员列表为空

4. **创建项目README**（使用 `edit_file` 工具）
   - 创建 `README.md`，包含项目基本信息和目录说明

#### 阶段三：更新项目索引

1. **读取项目索引**：使用 `read_file` 读取 `data-layer/projects/index.json`

2. **添加新项目记录**：使用 `edit_file` 或 `replace_in_file` 更新索引文件

3. **更新统计数据**：total +1, active +1

#### 阶段四：反馈用户

展示创建成功的信息：

```
✅ 项目创建成功！

项目信息：
- 项目ID：proj_20260212_001
- 项目名称：移动应用开发
- 项目描述：开发一款跨平台移动应用
- 工作空间路径：~/Documents/digital_projects/proj_20260212_001/
- 创建时间：2026-02-12 10:00:00

目录结构：
├── shared/           # 共享资源
│   ├── documents/    # 文档
│   ├── code/         # 代码
│   ├── design/       # 设计文件
│   └── config/       # 配置文件
├── members/          # 成员工作区
├── config/           # 项目配置
│   └── project.json
└── README.md         # 项目说明

下一步：
- 使用 "分配员工到项目" 功能添加团队成员
- 使用 "查看项目详情" 查看项目信息
```

### Step 2D：分配员工到项目

#### 阶段一：收集信息

1. **检查用户是否指定了项目**
   - 已指定 → 继续
   - 未指定 → 展示项目列表，询问选择

2. **检查用户是否指定了员工**
   - 已指定 → 继续
   - 未指定 → 调用Skill 1展示员工花名册，询问选择

3. **询问权限级别**（可选）
   - "请选择权限级别：1) 只读  2) 读写  3) 管理员（默认：读写）"

#### 阶段二：验证和检查

1. **验证项目存在**：在项目索引中查找项目

2. **验证员工存在**：调用Skill 1的"读取员工档案"接口

3. **检查员工是否已在项目中**：
   - 已在项目中 → 提示："员工'张三'已经是项目成员，是否要更新权限？"
   - 不在项目中 → 继续

#### 阶段三：更新项目数据

1. **更新项目配置文件**：
   - 使用 `read_file` 读取 `config/project.json`
   - 使用 `replace_in_file` 在 `members` 数组中添加新成员记录
   - 更新 `updatedAt` 时间戳

#### 阶段四：更新员工数据（调用Skill 1接口）

1. **调用"更新员工所属项目"接口**：
   - 参数：`employeeId`, `projectId`, `operation: "add"`
   - Skill 1会更新员工花名册和员工档案

#### 阶段五：创建项目引用文件

1. **在员工工作空间创建项目引用**：
   - 路径：`员工工作空间/项目引用/项目名称_ref.json`
   - 使用 `edit_file` 创建引用文件（参考数据结构设计）

2. **创建成员工作区目录**（可选）：
   - 路径：`项目工作空间/members/员工姓名/`
   - 用于存放该员工在项目中的个人工作文件

### Step 2G：归档/删除项目

#### 阶段一：收集信息和确认

1. **检查用户是否指定了项目**
   - 已指定 → 继续
   - 未指定 → 展示项目列表，询问选择

2. **询问操作类型**：
   - "请选择操作：1) 归档项目（保留数据）  2) 删除项目（永久删除）"

3. **安全确认**：
   - "此操作将影响 X 名成员，确认要继续吗？（输入项目名称以确认）"

#### 阶段二：执行归档操作

**如果选择归档**：

1. **更新项目状态**：
   - 在项目配置文件中设置 `status: "archived"`
   - 更新项目索引中的状态

2. **通知所有成员**：
   - 在每个成员的项目引用文件中添加 `archived: true` 标记

3. **更新统计数据**：
   - active -1, archived +1

**如果选择删除**：

1. **移除所有成员的项目关联**：
   - 调用Skill 1接口，移除所有成员的项目关联
   - 删除所有成员的项目引用文件

2. **删除项目工作空间**：
   - 使用 `terminal` 删除整个项目目录

3. **从项目索引中移除**：
   - 从 `data-layer/projects/index.json` 中删除项目记录

4. **更新统计数据**：
   - total -1, active -1

#### 阶段三：反馈用户


## 📋 工作准则

1. **数据访问规则**：
   - 只写 `data-layer/projects/` 目录
   - 只读 `data-layer/employees/` 目录
   - 通过Skill 1接口更新员工数据

2. **目录创建规范**：
   - 项目工作空间统一创建在 `~/Documents/digital_projects/`
   - 使用零依赖方式创建目录（terminal + mkdir）

3. **文件命名规范**：
   - 项目ID格式：`proj_YYYYMMDD_NNN`
   - 项目引用文件：`项目名称_ref.json`
   - 配置文件：`project.json`

4. **错误处理**：
   - 项目不存在时，提示用户并展示项目列表
   - 员工不存在时，调用Skill 1展示员工花名册
   - 权限不足时，提示用户并说明所需权限

5. **用户确认**：
   - 删除操作必须二次确认
   - 批量操作前展示影响范围

6. **日志记录**：
   - 所有关键操作记录到 `data-layer/logs/work_logs.json`
   - 记录内容：操作类型、操作者、时间、影响对象
